01  AI 활용 개요
케어 플랫폼 프로젝트는 3인 초보 개발팀이 15주 안에 공모전 제출용 앱을 완성해야 하는 도전적인 프로젝트입니다. 제한된 인원과 개발 경험을 AI로 보완하여, 기획부터 제출까지 전 과정에서 생산성을 극대화하는 것이 목표입니다.
핵심 원칙: Claude Max Plan을 중심축으로 활용하고, Claude로 커버할 수 없는 영역에만 최소한의 외부 도구를 보조적으로 사용합니다.

02  단계별 AI 활용 상세
2.1. 기획 단계 (1~2주차)
아이디어 구체화 · 문서 작성 · 시장 조사
Claude 코워크  |  메인 도구
PRD, 통합기획서, 경쟁 분석 문서 등을 .docx로 직접 생성하고 반복 수정합니다. 웹 검색 기능으로 시장 통계, 정책 동향, 경쟁 서비스 분석도 처리합니다.
	•	케어 플랫폼 PRD v2.1 작성 → 3역할 체계, AI 케어 가이드 기능 정의 반영
	•	통합기획서 v2.1 작성 → 15주 로드맵, 기능 우선순위 표, 기술 스택 정리
	•	요양원 이용 통계, 고령화 정책 동향 웹 리서치 → 근거 자료 수집
Claude 채팅  |  보조 도구
팀원 3명이 각자 브레인스토밍, 사용자 페르소나 정리, 기능 검증을 수행합니다.
	•	"보호자가 요양원에 부모님을 맡기고 가장 불안한 순간은 언제일까?" → 사용자 시나리오 도출
	•	AI 케어 가이드의 답변 톤 설계 → "전문적이지만 따뜻한 요양 상담사" 페르소나 정의
	•	경쟁 앱(오늘의요양, 얔케어 등) 기능 비교 분석 정리


2.2. 디자인 단계 (3~5주차)
와이어프레임 · UI 디자인 · UX 카피
Claude 코드  |  메인 도구
React + Tailwind CSS UI 컴포넌트를 프롬프트로 직접 생성합니다. 디자이너 없이도 코드 기반 프로토타이핑이 가능합니다.
	•	"케어 체크리스트 카드 UI 만들어줘" → 식사/투약/위생/활동 항목 체크 컴포넌트 생성
	•	"SOS 긴급 알림 모달 컴포넌트" → 풀스크린 빨간 알림 + 환자 정보 + 응답 버튼
	•	"AI 케어 가이드 채팅 화면" → 백과사전 모드/감정지원 모드 전환 UI
	•	"보호자 대시보드" → 오늘의 케어 요약, 체크리스트 완료율, 최근 알림 카드
Claude 코워크  |  보조 도구
UX 카피라이팅과 디자인 피드백을 담당합니다.
	•	역할별(보호자/의료진/환자) 버튼 텍스트, 안내 문구, 에러 메시지 작성
	•	온보딩 플로우 문구 설계: "가족의 케어를 함께 지켜보겠습니다"
	•	화면 설계안 기반 사용성 피드백 제공
Figma  |  외부 도구 (보조)
와이어프레임과 목업을 시각적으로 작업하는 데 사용합니다.
	•	화면 구성은 Claude 채팅으로 텍스트 설계 → Figma에서 시각화
	•	3역할(family/medical/patient) 별 화면 흐름 목업 작성
Midjourney / DALL-E  |  외부 도구 (보조)
Claude가 직접 생성할 수 없는 이미지 영역입니다.
	•	앱 내 온보딩 일러스트, 아이콘 세트
	•	공모전 발표자료용 비주얼 이미지


2.3. 개발 단계 (6~10주차)
프론트엔드 · 백엔드 · AI 기능 · 배포
Claude 코드  |  핵심 도구
프로젝트 전체 코드베이스를 이해한 상태에서 기능 단위 코드를 작성하고, 디버깅하고, 리팩토링합니다. git commit/PR까지 터미널에서 처리합니다.
	•	Supabase 테이블 생성 SQL + RLS 정책: family/medical/patient 3역할 기반 데이터 접근 제어
	•	F-01 케어 체크리스트: 의료진 입력 → 보호자 실시간 열람 전체 플로우 구현
	•	F-02 보호자 대시보드: Supabase Realtime 구독으로 실시간 데이터 반영
	•	F-03 SOS 긴급 알림: FCM 푸시 알림 + 미응답 시 자동 재알림 로직
	•	F-04 1:1 채팅: Supabase Realtime 기반 보호자-의료진 메시지 기능
	•	F-05 AI 일일 리포트: Claude API로 하루 케어 데이터 요약 → 보호자 자동 발송
	•	F-06 AI 케어 가이드: Claude API System Prompt 설계 + 백과사전/감정지원 모드 전환 로직
	•	F-07 역할 분기 + 인증: Supabase Auth + 역할별 라우팅 구현
	•	에러 발생 시: 에러 메시지 붙여넣기 → 원인 분석 + 즐시 수정
Claude API (Anthropic)  |  앱 내 탑재
케어 플랫폼 앱 자체에 Claude API를 연동합니다. Max Plan과 별도의 API 키가 필요하지만, 동일한 Anthropic 생태계로 일관성이 높습니다.
	•	AI 케어 가이드 메인: "장기요양등급 신청 절차" → System Prompt로 요양/건강 전문가 역할 부여
	•	AI 케어 가이드 서브: 대화 중 감정 신호 감지 → 자연스럽게 공감/위로 모드 전환
	•	AI 일일 리포트: 하루 케어 데이터 요약 → "오늘 요약: 식사 양호, 오후 산책 30분"
Claude 채팅  |  보조 도구
개발 중 학습과 문제 해결에 활용합니다.
	•	"Supabase RLS가 적용이 안 되는데 원인이 뭐야?" → 정책 코드 분석 + 해결책
	•	"React useEffect 무한 루프 해결" → 에러 패턴 설명 + 수정 방법 안내
	•	TypeScript 타입 에러 해결, Vercel 배포 이슈 트러블슈팅

05  AI 활용 운영 원칙
AI를 효과적으로 활용하기 위해 팀 전체가 따를 운영 원칙을 정리합니다.
원칙 1: AI 결과물은 반드시 검증한다
AI가 생성한 코드, 문서, 데이터는 팀원이 직접 리뷰하고 확인한 후 반영합니다. AI의 출력을 그대로 사용하지 않고, 이해한 뒤 반영하는 것이 핵심입니다.
원칙 2: 프롬프트를 기록한다
주요 AI 작업의 프롬프트와 결과물을 Notion에 기록합니다. 공모전 심사 시 "AI를 어떻게 활용했는가"를 구체적으로 설명할 수 있는 근거 자료가 됩니다.
원칙 3: 도구를 최소화한다
Claude Max Plan으로 커버할 수 있는 영역은 모두 Claude로 통일하고, 외부 도구는 Figma(목업), Midjourney(이미지), Notion(프로젝트 관리) 3개로 제한합니다. 도구 난립을 방지하고 학습 부담을 줄입니다.
원칙 4: AI로 학습한다
초보 팀이므로 AI가 생성한 코드와 구조를 통해 실전적인 개발 학습을 병행합니다. "이 코드가 왜 이렇게 동작하는지 설명해줘" 같은 후속 질문을 통해 코드 이해도를 높입니다.

AI를 도구로, 사람을 중심으로.

AI 가이드가 보호자 질문에 대한 답변 초안 생성 → 의료진이 한 번 확인만 하면 되는 "AI 보조 응답" 시스템 → 의료진 업무 부담 최소화하면서 응답 속도 향상

Klara — AI Assistant 업무 자동화
개요: Klara는 앱 설치 없이 2FA만으로 접근 가능한 의료 소통 플랫폼으로, AI Assistant가 반복 업무를 자동화합니다.
항목
내용
AI 기술
AI Assistant (규칙 기반 + ML 자동화)
작동 방식
모든 채널(전화, 문자, 이메일, 웹) → 통합 인박스 → AI가 자동 분류·라우팅 반복 질문 자동 답변, 예약 확인 자동 발송
효과
의료 스태프의 전화 응대 시간 50%+ 감소
하루안부 적용
• 웹 우선 접근(앱 설치 불필요) → 시니어/보호자 진입 장벽 최소화 • 통합 채널 설계 → 카카오톡, 웹, 앱 어디서든 동일한 경험 • AI 자동 분류 → 긴급/일반/행정 메시지 자동 라우팅

OhMD — AI 에이전트 "Nia" 자동 라우팅
개요: OhMD의 AI 에이전트 "Nia"는 환자 메시지를 자동으로 분석하여 적절한 담당자에게 라우팅합니다.
항목
내용
AI 기술
AI 에이전트 "Nia" (자연어 이해 + 자동 라우팅)
작동 방식
환자 메시지 수신 → Nia가 내용 분석 → 긴급도/유형 판단 → 적절한 의료진에게 자동 전달 단순 문의는 AI가 직접 답변
효과
메시지 분류 자동화, 긴급 메시지 놓치는 비율 감소
하루안부 적용
• AI 가이드에 "Nia" 방식 라우팅 적용   - 보호자 메시지 → AI가 긴급도 분류 (SOS / 일반 / 행정)   - SOS 메시지 → 즉시 의료진 알림   - 일반 질문 → AI 가이드가 1차 답변 후 필요시 의료진 연결 • F-04 AI 가이드의 핵심 기능으로 반영

채널톡 ALF — 3단계 에스컬레이션 → 하루안부 적용
개요: 채널톡의 AI 봇 ALF는 3단계 에스컬레이션 구조로 고객 문의를 처리하며, 이 패턴이 하루안부 AI 가이드에 가장 직접적으로 적용 가능합니다.
단계
채널톡 ALF
하루안부 AI 가이드 적용
1단계 FAQ 자동
FAQ 데이터베이스에서 즉시 답변 (예: 영업시간, 반품 정책)
요양 일반 정보 FAQ 자동 답변 (예: 면회 시간, 식단 안내, 요양 등급 설명)
2단계 AI 맞춤
AI가 대화 맥락 파악 후 개인화 답변 고객 이력 참고하여 맞춤 제안
AI가 환자 상태·이력 기반 맞춤 답변 (예: "어머니 혈압이 어제보다 높은데, 담당 간호사에게 확인해 볼까요?")
3단계 전문가 연결
해결 불가 시 상담원에게 자동 전달 대화 맥락 함께 전달
의료 판단 필요 시 의료진에게 자동 연결 AI가 정리한 맥락 + 환자 상태 함께 전달

핵심 적용 원칙:
1. AI는 "1~2단계"에서 보호자의 80%+ 질문을 처리하여 의료진 업무 부담 최소화
2. 3단계 전환 시 AI가 대화 요약을 함께 전달하여 의료진이 맥락 파악에 시간 소모하지 않도록 함
3. 모든 단계에서 "지금 AI가 답변 중입니다" / "의료진에게 연결 중입니다" 등 상태를 명확히 표시
4. 의료 관련 답변에는 반드시 "이 정보는 참고용이며, 정확한 진단은 의료진에게 문의하세요" 면책 문구 포함
