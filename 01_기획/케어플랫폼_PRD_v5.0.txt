1. 서비스 개요
1.1 핵심 컨셉
케어 플랫폼은 장기 돌봄이 필요한 환자(요양원 입소 노인, 장기입원 아동, 재가 중증 환자)의 보호자가 경험하는 소통 단절 문제를 해결하는 케어 커뮤니케이션 플랫폼입니다.
보호자 — 의료진(의사/간호사) — 환자 3개 역할을 하나의 앱에서 연결합니다. 역할에 따라 완전히 다른 UI를 제공하지만 앱은 1개입니다.

1.2 플랫폼 전략: 웹 우선 개발
의료진(의사/간호사)은 주로 PC에서 작업합니다. 처방전 작성, 진료기록 확인, 체크리스트 입력 등의 업무를 핸드폰이나 아이패드가 아닌 데스크톱 컴퓨터로 수행합니다. 따라서 웹을 먼저 개발하고, 이후 모바일 반응형 + PWA로 확장하는 전략을 채택합니다.

1.4 시장 배경 및 기회
	•	2024년 한국 초고령사회 공식 진입 — 65세 이상 1,024만 명(20%) 돌파
	•	장기요양 이용자 2023년 110만 명 → 2027년 145만 명 전망
	•	전국 요양원 4,640개소, 요양원 68%가 면회 제한 규정
	•	보호자 심각한 스트레스 경험 2022년 36.0% → 2024년 46.3% 급증
	•	요양원 정보 탐색이 어려운 보호자를 위한 요양·건강 정보 통합 플랫폼 부재
	•	가족-환자-의료진 실시간 연결 플랫폼은 국내 시장에 존재하지 않음

4. 주요 기능 수락 조건 (Acceptance Criteria)
각 Level 1 기능의 완료 기준을 명확히 정의합니다.
F-01 케어 체크리스트
	•	의료진이 PC 웹에서 환자별 식사·투약·위생·활동 항목을 탭으로 체크할 수 있다
	•	체크 시 타임스탬프가 자동 저장된다
	•	체크 입력 후 3초 이내에 보호자 대시보드에 반영된다 (Supabase Realtime)
F-02 보호자 대시보드
	•	로그인 시 연결된 환자의 오늘 케어 현황이 카드형으로 표시된다
	•	체크리스트 변경 시 페이지 새로고침 없이 실시간 업데이트된다
F-03 SOS 긴급 알림
	•	의료진이 PC에서 SOS 버튼을 누르면 30초 이내에 보호자에게 푸시 알림이 발송된다
	•	상황 메모를 입력할 수 있다
	•	알림 발송 → 확인 → 대응 완료까지 타임라인이 자동 기록된다
F-04 1:1 채팅
	•	보호자와 담당 의료진이 텍스트 메시지를 주고받을 수 있다
	•	사진 첨부가 가능하다 (Supabase Storage)
	•	새 메시지 수신 시 푸시 알림이 발송된다
F-05 AI 일일 케어 리포트
	•	매일 저녁 시간에 당일 체크리스트 데이터를 AI가 자연어로 요약한다
	•	리포트가 보호자 앱에 푸시 알림으로 발송된다
F-06 AI 케어 가이드 (핵심 차별화)
▶ 메인: 요양·건강 정보 백과사전
	•	요양원 정보: 입소 절차, 비용 구조, 장기요양 등급 신청 방법, 복지 제도 안내
	•	건강 정보: 노인성 질환 기초 지식, 투약 주의사항, 재활 운동, 영양 관리
	•	의사결정 지원: 객관적 정보 제공으로 보호자의 판단 도움
	•	대화 예시: "장기요양등급 신청은 어떻게 하나요?" → 신청 절차·필요 서류·소요 기간 상세 안내
▶ 서브: 감정 지원
	•	대화 흐름에서 감정적 신호 감지 시 자연스럽게 공감·위로 모드로 전환
	•	피로도 누적 시 전문 상담 기관 정보 안내
	•	법적 프레이밍: "심리 치료" X, "의료 진단" X → "요양·건강 정보 안내 및 감정 체크인 서비스"
F-07 역할 분기 + 인증
	•	카카오 또는 구글 소셜 로그인으로 인증
	•	로그인 후 DB의 role에 따라 보호자/의료진/환자 화면으로 자동 분기
	•	타 역할 화면에는 접근할 수 없다
F-08 시간표 (★ NEW)
환자의 하루 일과와 의료진의 근무/케어 스케줄을 체계적으로 관리하는 기능입니다.
▶ 환자 일과표 (환자 + 보호자 열람)
	•	하루 일과: 기상 → 아침 식사 → 오전 투약 → 재활운동 → 점심 → 산책 → 저녁 → 취침
	•	환자는 모바일/태블릿에서 큰 글자로 오늘 일과 확인
	•	보호자는 환자의 일과표를 열람하여 어떤 활동을 하는지 파악
▶ 의료진 근무/케어 스케줄 (의료진 PC 웹)
	•	근무 시간표: 오늘의 담당 환자 목록 + 케어 스케줄
	•	환자별 케어 타임라인: 어떤 환자에게 몇 시에 무엇을 해야 하는지 한눈에 파악
	•	체크리스트와 연동: 스케줄 항목 완료 시 체크리스트 자동 반영
F-09 처방전·진료기록 + 의료 타임라인 (★ NEW)
의료진이 PC에서 처방전과 진료기록을 입력·관리하고, 보호자와 환자가 권한에 따라 열람할 수 있다. 접근을 시간순 타임라인으로 통합한다.
▶ 의료진 (PC 웹 — 입력 + 관리)
	•	처방전 입력: 약명, 용량, 투약 시간, 주의사항, 처방일
	•	진료기록 입력: 진료일, 진단명, 소견, 추후 계획, 담당의
	•	이력 관리: 환자별 처방전/진료기록 타임라인 조회
▶ 보호자 (모바일 웹 — 열람)
	•	현재 처방전 열람: 약명, 용량, 투약 시간 확인
	•	진료기록 요약 열람: 진단명, 소견 요약 (상세 내용은 의료진 판단에 따라 공개 범위 설정)
▶ 환자 (모바일/태블릿 — 제한적 열람)
	•	본인 처방전 확인: 약명, 투약 시간 (큰 글자 UI)
▶ 권한 및 보안
	•	Supabase RLS로 역할별 접근 범위 강제: 의료진 = 전체, 보호자 = 요약, 환자 = 본인 처방전만
	•	의료 정보 AES-256 암호화 저장
▶ 의료 타임라인 (진료·처방·검사 통합 뜀)
진료기록, 처방 변경, 검사 기록을 시간순 타임라인으로 통합하여 보여주는 뜀입니다. F-08(시간표)과 F-09(처방전·진료기록)의 데이터를 하나의 흐름으로 시각화합니다.
	•	보호자: 환자의 최근 의료 이력을 시간순으로 한낈에 확인 (예: 3/14 정기검진 → 3/12 혹애약 용량 조정 → 3/10 현액검사 정상)
	•	의료진: PC에서 환자별 전체 의료 이력 타임라인 조회 + 새 기록 추가
	•	환자: 본인 의료 이력 타임라인 열람 (큰 글자 UI)
	•	타임라인 항목 유형: 진료(diagnosis), 처방(prescription), 검사(examination)
	•	각 항목 클릭 시 상세 정보 폄처보기
