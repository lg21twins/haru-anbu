## 1. 브랜드 이름과 의미

### 서비스명: 하루안부 (Haru Anbu)

**어원 분석**

| 어원 | 의미 |
|------|------|
| 하루 (日) | 하루, 오늘, 매일 — 일상의 단위 |
| 안부 (安否) | "편안함과 그렇지 않음을 묻는 것" — 안녕을 묻는 행위 |

**합성 의미**: "매일 안부를 묻다" — 전화 없이도 오늘 하루를 알 수 있는 서비스

**슬로건**: 오늘 하루도, 안녕하셨습니다

슬로건은 두 가지로 읽힌다:
1. "오늘 하루도 안녕하셨습니다" — 보호자에게 전하는 안심 보고
2. "오늘 하루도, 안녕하셨습니까?" — 매일 반복되는 따뜻한 안부 인사

이 이중성이 하루안부의 본질이다. **매일의 정보 전달**이자 **인간적인 연결**.

## 2. 브랜드 포지셔닝

### 브랜드 퍼스낼리티

> "따뜻하지만 전문적인 — 안심을 디자인하다"

| 속성 | 설명 | 구체적 표현 |
|------|------|-----------|
| 따뜻한 (Warm) | 가족에게 보내는 편지 같은 감정 | 둥근 모서리, 따뜻한 색조, ~요 체 어미 |
| 전문적 (Professional) | 의료진이 신뢰할 수 있는 시스템 | 명확한 정보 위계, 데이터 정확성 |
| 안심 (Reassuring) | "괜찮아, 오늘도 잘 계셔" 의 감정 | 안심 블루, 완료 그린, 충분한 여백 |
| 소프트 (Soft) | 병원·키오스크 느낌을 배제 | 카드 레이아웃, 라인 일러스트, 소프트 그림자 |

### 브랜드 포지셔닝 매트릭스

```
               따뜻함
                 ↑
  캐주얼 ←──────┼──────→ 전문적
          하루안부 ●
                 ↓
               차가움
```

- 기존 의료 서비스: 전문적 + 차가움 (오른쪽 하단)
- 일반 케어 매칭 앱: 캐주얼 + 따뜻함 (왼쪽 상단)
- 하루안부: **전문적이면서 따뜻한** (오른쪽 상단) — 유일한 포지션

## 4. 타이포그래피 시스템

### 4.1 브랜드 서체

**한국어**: Pretendard Variable (CDN: orioncactus/pretendard)
**영어/숫자**: Pretendard 내장 라틴 (자동 적용)
**폴백**: -apple-system → Helvetica Neue → sans-serif

```css
font-family: 'Pretendard Variable', 'Pretendard', -apple-system, 'Helvetica Neue', sans-serif;
```

**Pretendard 선택 이유**:
- 한국어 가독성 최적화 (완성형 + 조합형 전체 지원)
- Variable Font — 단일 파일로 모든 굵기 지원
- 소숫점 웨이트 가능 (100~900)
- 서울서체·애플SD산돌고딕과 달리 상업 무료

### 4.2 타이포그래피 스케일

| 역할 | 크기 | 굵기 | CSS 토큰 | 사용처 |
|------|------|------|---------|-------|
| Display | 32px | Bold (700) | `--text-display` | 화면 대제목 (메인 숫자, 환영 메시지) |
| Title | 22px | SemiBold (600) | `--text-title` | 섹션 제목, 모달 헤더 |
| Headline | 17px | Medium (500) | `--text-headline` | 카드 제목, 행 제목 |
| Body | 16px | Regular (400) | `--text-body` | 일반 본문 |
| Body Large | 18px | Regular (400) | `--text-body-large` | 환자 화면 본문 (고령자 가독성) |
| Callout | 14px | Regular (400) | `--text-callout` | 보조 정보, 설명 |
| Caption | 13px | Regular (400) | `--text-caption` | 타임스탬프, 메타정보 |
| Mini | 11px | Regular (400) | `--text-mini` | 최소 텍스트, 배지 |

**줄 간격**
- Tight (1.2): 제목, 헤드라인
- Normal (1.4): 본문 기본값
- Relaxed (1.6): 환자 화면, AI 리포트 본문

### 4.3 환자 화면 접근성 타이포그래피

환자 화면(`data-role="patient"`)은 일반 스케일보다 2px 크게 운용:
- 본문: 18px (Body Large)
- 제목: 24px+
- 최소 터치 영역: 56px 이상

## 7. 브랜드 보이스 & UX 라이팅

### 7.1 보이스 원칙

**한 줄 정의**: 따뜻하지만 정확하다. 가족에게 말하듯, 하지만 정보는 놓치지 않는다.

| 속성 | 설명 |
|------|------|
| 친근함 | -요, -어요 종결어미. "어머님", "보호자님" 호칭 |
| 정확함 | 수치·시간은 명확하게. 모호한 표현 배제 |
| 간결함 | 한 문장으로 핵심 전달. 긴 설명 지양 |
| 안심감 | 긍정적 프레이밍. "완료"보다 "잘 됐어요" |

### 7.2 UX 라이팅 예시

| 상황 | 사용하지 않는 표현 | 사용하는 표현 |
|------|-----------------|------------|
| 케어 완료 알림 | "투약 처리 완료" | "오전 투약이 완료되었어요 ✅" |
| AI 리포트 | "일일 보고서 생성됨" | "오늘 어머님은 이렇게 보내셨어요" |
| SOS 알림 | "긴급 상황 발생" | "어머님 관련 긴급 알림이 도착했어요" |
| 빈 상태 (Empty State) | "데이터 없음" | "아직 오늘의 기록이 없어요" |
| 오류 | "오류 발생" | "잠시 연결이 불안정해요. 다시 시도해볼게요" |
| AI 가이드 인사 | "무엇을 도와드릴까요?" | "안녕하세요, 무엇이든 물어보세요 😊" |
| 로딩 | "처리 중..." | "잠시만 기다려주세요" |
| 완료 | "성공" | "완료됐어요!" |

### 7.3 피해야 할 표현

- 의료 용어 과다: "투약", "처치", "기록" → "약", "케어", "오늘 일과"
- 기계적 어미: "~하십시오", "~바랍니다" → "~해보세요", "~해주시겠어요?"
- 불안 자극: "주의", "경고", "실패" 과다 → 필요한 경우에만 제한적으로

## 8. 아이콘 & 일러스트레이션 원칙

### 8.1 아이콘 스타일 가이드

**스타일**: Line icon (선형 아이콘)
- 선 두께: 1.5px ~ 2px
- 모서리: rounded cap, rounded join (각진 끝선 배제)
- 컬러: 역할 primary 또는 text-secondary
- 활성 상태: filled + primary 컬러
- 비활성 상태: line + text-secondary

**레퍼런스 라이브러리**: Lucide Icons (권장) / Heroicons / Phosphor Icons
- 선택 기준: rounded 스타일, 일관된 선 굵기

### 8.2 일러스트레이션 원칙

**스타일**: 미니멀 라인 일러스트 (색 면 최소화)

| 원칙 | 설명 |
|------|------|
| 선 중심 | 복잡한 색 면보다 깔끔한 선 위주 |
| 따뜻한 색조 | 역할 컬러 또는 accent 색 포인트 1개 |
| 인물 다양성 | 고령자·보호자·의료진이 자연스럽게 등장 |
| 적절한 여백 | 빽빽하지 않게, 여백이 일러스트의 일부 |

**사용 위치**:
- Empty State (데이터 없음 상태)
- 온보딩 화면 (3개 슬라이드 배경)
- AI 리포트 상단 포인트 일러스트

**사용하지 않는 것**:
- 사진 스톡 이미지 (병원·의료 관련 포함)
- 복잡한 3D 오브젝트
- 캐릭터 지향 이모지형 일러스트

## 9. 모션 & 인터랙션

### 9.1 트랜지션 토큰

| 토큰 | 값 | 사용처 |
|------|-----|-------|
| `--transition-fast` | 150ms ease | 버튼 호버, 토글 |
| `--transition-normal` | 250ms ease | 카드 펼침, 탭 전환 |
| `--transition-slow` | 350ms ease | 모달 등장, 바텀시트 |

### 9.2 진입 애니메이션

```css
@keyframes fadeInUp {
  from { opacity: 0; transform: translateY(16px); }
  to   { opacity: 1; transform: translateY(0); }
}
```

- 카드 리스트 진입: fadeInUp + 50ms 순차 딜레이
- 모달/바텀시트: slideUp (translateY 100% → 0)
- 탭 전환: opacity fade (150ms)

**원칙**: 모션은 정보 전달을 돕는 것. 과도한 모션은 브랜드 신뢰감을 떨어뜨린다.

| 텍스트 / 배경 | WCAG AA 이상 (4.5:1) |

| UI 컴포넌트 / 배경 | 3:1 이상 |

### 10.2 환자 화면 특별 규칙

- 최소 폰트: 18px (일반 16px → 2px 상향)
- 최소 터치 영역: 56px × 56px
- 버튼 텍스트: SemiBold 이상
- 탭 레이블: 항상 표시 (아이콘만으로 구성 불가)
- 색상 단독 의존 금지: 아이콘·텍스트·모양으로 중복 표현

### 10.3 포커스 & 상태 표현

- 포커스 링: `2px solid var(--primary)`
- Disabled: opacity 0.4 + pointer-events: none
- 로딩: 스켈레톤 UI (회색 블록) 우선, 스피너 보조

## 11. DO / DON'T

### 브랜드 무드 DO
- 가족에게 보내는 **안심 편지** 같은 앱
- **카드 기반** 레이아웃 — 정보를 카드 단위로 분리
- **둥근 모서리** (border-radius: 14px+), 부드러운 그림자
- **여백 넉넉** — "덜 보여주는 용기"
- **라인 일러스트** 포인트 — 간결하고 따뜻
- 부드러운 말투 — "~하셨습니다", "~해보세요"

### 브랜드 무드 DON'T
- 차가운 **병원 차트** 느낌
- 의료 **키오스크** 느낌
- 날카로운 직선, 각진 UI (radius 0~4px 배제)
- 빽빽한 데이터 나열
- 스톡 사진 (특히 병원·의료 관련)
- 딱딱한 말투 — "~하십시오", "~바랍니다"
- 빨간색(critical) 과다 사용
