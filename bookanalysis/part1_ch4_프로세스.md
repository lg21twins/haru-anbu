# 1부 4장: UX·UI 디자인 프로세스

## 핵심 요약

UX·UI 디자인 프로세스는 사용자의 요구와 경험을 중심으로 제품이나 서비스를 설계하고 개발하는 일련의 체계적이고 반복적인 단계를 의미한다.

## 1. 프로세스의 정의와 역할

### 프로세스란?
- **입력(Input) → 주요 단계(사고, 아이디에이션 등) → 산출(Output)** 의 구조
- 품질 보증(Quality Assurance): 프로세스의 각 단계가 체계적이고 반복적일수록 효과적
- **피드백(Feedback)** 루프를 통해 결과를 평가하고 프로세스를 개선

### 프로세스의 기본 구성 요소
- **마일스톤(Milestone)**: 프로세스 내 주요 검토/전환 지점
- **메서드(Method)**: 각 단계에서 사용되는 구체적인 방법론
- **매니지먼트(Management)**: 프로세스 전체의 관리와 조정

### UX·UI 디자인 프로세스의 핵심
- **사용자 중심의 프로세스**를 지향
- 사용자 리서치 → 사용자 요구 이해 → 디자인 → 프로토타이핑 → 테스트 → 개선의 반복

## 2. 디자인 프로세스 유형

### 주요 프로세스 모델
| 모델 | 특징 |
|------|------|
| **폭포수(Waterfall)** | 순차적, 선형적 접근 |
| **린(Lean)** | 최소화된 자원으로 빠른 프로토타입 제작 |
| **애자일(Agile)** | 반복적이고 점진적인 개발 |
| **더블 다이아몬드(Double Diamond)** | 2005년 영국 디자인 카운슬(Design Council)에서 제안. 발산(Discover) → 수렴(Define) → 발산(Develop) → 수렴(Deliver) |
| **디자인 씽킹(Design Thinking)** | 공감 → 정의 → 아이디에이션 → 프로토타입 → 테스트 |
| **구글 디자인 스프린트(Google Design Sprint)** | 5일간 집중적으로 문제 해결 |

### 프로세스의 공통 특징
- 사용자와 함께 **반복적(Iterative)**으로 진행
- **정량적 + 정성적** 방법을 병행
- 비즈니스 목표와 사용자 요구의 **균형** 추구

## 3. UX·UI 프로세스 기본 단위의 확장적 구성

### 8단계 프로세스 모델

| 단계 | 이름 | 설명 |
|------|------|------|
| 1단계 | **발견(Discover)** | 사용자 리서치와 시장 조사를 통해 문제를 발견 |
| 2단계 | **정의(Define)** | 사용자의 요구를 구체적으로 정의, 사용자 페르소나 작성 |
| 3단계 | **구조(Structure)** | 고객 여정 맵핑, 사용자 경험의 전체적 구조를 설계, 정보 구조(IA) |
| 4단계 | **디자인(Design)** | 비전과 디자인의 외적 형태 결정, 시각적 요소 결정 |
| 5단계 | **프로토타입(Prototype)** | 실제로 사용할 수 있는 형태의 프로토타입 제작 |
| 6단계 | **테스팅(Testing)** | 사용자 테스트로 문제점 분석 및 개선, 피드백 수집 |
| 7단계 | **개선(Iteration)** | 테스트 결과와 사용자 피드백을 반복적으로 반영하여 개선 |
| 8단계 | **배포(Deployment)** | 개발 및 출시, 출시 후 사용자 지원과 지속적 업데이트 |

### 각 단계의 핵심 포인트
- **발견 → 정의**: 문제를 올바르게 정의하는 것이 해결의 출발점
- **구조 → 디자인**: 정보 구조를 먼저 설계한 후 시각적 디자인을 적용
- **프로토타입 → 테스팅 → 개선**: 반복적인 피드백 루프가 핵심
- **배포**: 출시 후에도 지속적인 사용자 지원과 업데이트 필요

## 핵심 인사이트

1. UX·UI 프로세스는 **선형이 아닌 반복적(Iterative)** 구조
2. **더블 다이아몬드** 모델이 가장 대표적: 발산과 수렴을 2번 반복
3. 8단계 프로세스: 발견 → 정의 → 구조 → 디자인 → 프로토타입 → 테스팅 → 개선 → 배포
4. 각 단계마다 **피드백과 반복**이 내재되어 있음
5. 프로세스는 **비즈니스 문제를 해결하는 체계적 접근법**
