# 2부 7장: 인사이트 도출

## 핵심 요약

UX 디자인 리서치를 통해 수집된 데이터에서 의미 있는 인사이트를 도출하는 것이 이 장의 핵심이다. 어피니티 다이어그램, ER 매트릭스, 포지셔닝 맵, 다이어그램(UML 포함) 등 다양한 분석 및 시각화 도구를 활용하여 리서치 데이터를 체계적으로 정리하고 인사이트를 발견한다.

## 1. 분석 방법에 대한 개괄

### 분석의 기본 원리
- **분석(Analysis)**: 하나하나의 발견을 모아 큰 그림을 구성
- **리서치 데이터**를 정성적·정량적으로 분류하여 패턴과 의미 도출
- **질적 조사**: 인터뷰, FGI, 관찰 등 / **양적 조사**: 설문, 데이터 분석 등

### 주요 분석 도구
| 텍스트 리서치 | 다이어그램 |
|------|------|
| AEIOU 프레임워크 | 퍼소나 |
| 어피니티 맵 | 고객 여정 지도 |
| 포지셔닝 맵 | 로커스/서비스 브랜드 |
| 공감 | 포커셔닝 맵 |
| 갈래/마인드맵 | CX 매트릭스 |

## 2. 인사이트 분석 방법

### AEIOU 프레임워크
- **A(Activities)**: 사람들의 행동/활동
- **E(Environments)**: 환경, 공간적 맥락
- **I(Interactions)**: 사용자 간 또는 사용자-객체 간 상호작용
- **O(Objects)**: 제품, 도구, 서비스 등 객체
- **U(Users)**: 사용자의 특성, 역할, 감정
- POEMS와 유사: People, Objects, Environments, Messages, Services

### ER 매트릭스 (Entity Relation Matrix)
- **주어(Entity)와 객체(Entity)** 또는 **개체 관계**를 매트릭스로 표현
- Charles Owen이 시카고 일리노이공과대학에서 개발
- 요소와 관계를 **정량적이고 체계적**으로 파악
- 다이어그램 형식으로 배치하여 **관계의 강도** 시각화

### 어피니티 맵 (Affinity Map)
- **어피니티 다이어그램(Affinity Diagram)**: KJ기법이라고도 함
- 1960년대 일본의 문화인류학자 가와키타 지로(川喜田二郎)가 개발
- **포스트잇에 개별 아이디어/발견 기록** → 유사성에 따라 **그룹핑** → 패턴 도출
- Open card sorting과 유사

### 어피니티 노트 만들기
1. 관찰한 내용을 **하나씩 포스트잇에 기록**
2. 주 참가자의 의견과 관찰자의 의견 구분
3. VoC(Voice of Customer) 등 참가자의 직접 인용 포함
4. 하나의 포스트잇에 **하나의 내용**만 기록

### 그룹핑
- 어피니티 노트를 **유사 기준**에 따라 그룹화
- 프로젝트 목표 및 전략에 맞춰 카테고리 구성
- **의미 추출**: 그룹핑된 데이터에서 종합적 의미와 패턴 발견

## 3. 포지셔닝 맵과 다이어그램

### 포지셔닝 맵 (Positioning Map)
- **2개 축**을 기준으로 경쟁 관계에 있는 제품/서비스를 **비교 분석**
- 축 선정이 가장 중요: 분석 목적에 맞는 **의미 있는 축** 선택
- 나열 요소: 아이디어, 나라, 제품명, 브랜드 등
- **2×2 포지셔닝**: x축과 y축의 교차점에서 강도와 위치 파악
- 이항 대립적 요소를 축에 배치하여 **가장 강조되어야 할 영역** 도출

### 다이어그램 (Diagram)
- **다이어그램**: 보통 벤다이어 낮은 활용도지만, UX 디자인에서는 매우 중요
- 실상 UX 디자인에서 사용되는 다이어그램은 **의미를 전달하는 도구**
- 데이터 관계, 상호작용, 프로세스를 시각적으로 표현

### UML (Unified Modeling Language)
- 1990년대 미국 소프트웨어 개발 중심으로 등장
- **Grady Booch, James Rumbaugh, Ivar Jacobson** 세 개발자 참여
- 13가지 이상의 다이어그램 유형: 클래스, 시퀀스, 유스케이스, 스테이트, 액티비티 등

### 유스케이스 다이어그램 (Use Case Diagram)
- 시스템의 **기능을 사용자 시점**에서 표현
- **액터(Actor)**: 시스템 외부에서 상호작용하는 사람/시스템
- **유스케이스(Use Case)**: 시스템이 제공하는 기능 단위
- 시스템의 경계와 범위를 명확히 정의

### 스테이트 다이어그램 (State Diagram)
- 시스템의 **상태 변화**를 시각적으로 표현
- ATM 스크린 상태 변화 등 구체적 사례 적용
- OFF → 대기상태 → 암호 입력 → 본인확인 → 계좌 선택 → 거래 → 종료

### 시퀀스 다이어그램 (Sequence Diagram)
- **특정 행동이 어떤 단계를 거쳐 어떻게 상호작용하는지** 표현
- 객체(Object), 생명선(Life line), 상호작용(Interaction) 포함
- 레스토랑에서 고객-웨이터-주문 시스템 간의 상호작용 사례
- Man-machine / Machine-machine Interaction 구분

### 플로차트와 워크플로 다이어그램
- **플로차트(Flowchart)**: 프로그래밍 절차를 도식화
- **워크플로 다이어그램(Workflow Diagram)**: UML에 속하지 않지만 유사한 구조
- 레스토랑 식사 과정을 플로차트로 표현한 사례

## 핵심 인사이트

1. **AEIOU 프레임워크**로 리서치 데이터를 체계적으로 분류하고 정리
2. **어피니티 맵(KJ기법)**은 정성적 데이터를 그룹핑하여 패턴을 발견하는 핵심 도구
3. **포지셔닝 맵**은 경쟁 분석과 전략 수립에 가장 직관적인 시각화 도구
4. **UML 다이어그램**(유스케이스, 스테이트, 시퀀스)은 시스템 설계와 사용자 흐름을 구체적으로 표현
5. 인사이트 도출의 핵심은 **데이터를 의미 있는 패턴으로 연결**하는 것
