────────────────────────────────────────────────
1. 3C 분석 (Customer · Competitor · Company)
────────────────────────────────────────────────
3C 분석은 고객(Customer), 경쟁자(Competitor), 자사(Company)의 세 축으로
서비스 전략의 핵심 방향을 도출하는 프레임워크다 (교재 Part 2 Ch.8 참조).


──────────────────
1-A. Customer (고객)
──────────────────
타겟: 요양원 입소 환자의 보호자 · 의료진 · 환자 3자

세그먼트 1 — 보호자 (1차 타겟 ★★★)
  규모      : 장기요양 인정자 117만 명의 직계가족 (추정 약 200만 명+)
  대표 인물  : 김미영 (52세, 서울 강남구 직장인, 요양원 어머니)
  핵심 니즈  : "오늘 엄마가 괜찮은지 매일 알고 싶다"
  행동 패턴  : 퇴근 후 18~22시에 요양원 전화 → 연결률 60% → 불안 증가
  지불 의향  : 월 9,900~19,900원 프리미엄 구독 (토스, 배민 수준 허들)
  얼리어답터 : 매일 전화하는 보호자 → 불안 강도가 전환율과 직결

세그먼트 2 — 의료진 (2차 타겟 ★★★)
  규모      : 전국 요양원 4,640개소 × 평균 간호사 3~5명 ≈ 약 5~7만 명
  대표 인물  : 박지현 (34세, 경기 성남 요양원 간호사 7년차, 담당 15명)
  핵심 니즈  : "보호자 전화 10통 줄이고 환자 돌봄에 집중하고 싶다"
  행동 패턴  : 수기 기록 1시간+ / 일, 3교대 인수인계 구두 반복
  얼리어답터 : 수기 기록에 1시간 이상 쓰는 간호사

세그먼트 3 — 환자 (서브드 타겟 ★★☆)
  규모      : 장기요양 인정자 117만 명 (2024 기준)
  대표 인물  : 김순자 (78세, 장기요양 3등급, 치매 초기, 고혈압)
  핵심 니즈  : "가족과 연결되어 있다는 안심이 필요하다"
  행동 패턴  : 스마트폰 기본 통화만 가능, 글자 작으면 포기
  UX 원칙   : 최소 탭 3개, 글자 18pt+, 버튼 48×48px+

고객 인사이트 요약
  - 3자가 모두 사용할수록 네트워크 가치 증가 (멀티사이드 플랫폼)
  - 보호자의 "불안" = 가장 강력한 구매 동인
  - 의료진의 "업무 감소" = 도입 설득의 핵심
  - 환자는 UX 접근성이 진입 장벽 결정


──────────────────────
1-B. Competitor (경쟁자)
──────────────────────
국내 요양 관련 서비스는 "입소 전 매칭" 또는 "시설 운영 B2B"에 집중.
하루안부가 목표하는 "입소 후 3자 실시간 소통" 영역은 사실상 공백 상태.

국내 경쟁사 비교

서비스         유형              핵심 기능                           케어플랫폼과의 관계
──────────────────────────────────────────────────────────────────────────────────
케어닥         B2C 매칭          AI 매칭, 간병인 프로필, 24시간 긴급   보완 (입소 전 단계)
케어네이션     B2C AI 매칭       자격증 자동분류, 신분인증, 리포트      보완 (입소 전 단계)
또하나의가족   B2C 시설탐색      요양원 검색, 복지용구 쇼핑             보완 (입소 전 단계)
케어포         B2B SaaS         기관 업무관리, 급여기록, 청구           부분 경쟁 (기관 관리)
엔젤시스템     B2B SaaS         운영관리, 보호자 공지                  부분 경쟁 (단방향)
케어로         B2C 모니터링     보호자 관점 하루 모니터링 UI            직접 경쟁 (기능 단순)
스마트장기요양 공공 B2B         재가서비스 일정, 전자태그               타겟 상이

글로벌 경쟁사 비교 (벤치마킹 대상)

서비스         국가   핵심 특징                              차별점 vs 하루안부
──────────────────────────────────────────────────────────────────────────────
Birdie         영국   가족 앱 + 케어플랜 통합, 1000+ 기관    한국 제도 미지원, 영어 전용
Caily          미국   Circle 3자 소통, AI 건강 인사이트       한국 시장 미진출
PointClickCare 미국   북미 최대 요양 플랫폼, 가족 포탈       한국 규제/제도 미대응
Carely         미국   가족 간 케어 협업, 사진/스토리 공유    의료진 연결 기능 약함

핵심 경쟁 인사이트
  - 국내: "입소 후 양방향 소통 + 기능 다양" 사분면 완전 공백
  - 해외: Birdie·Caily가 근접하지만 한국 시장 미진출 + 제도 미대응
  - 하루안부: 한국어 + 한국 장기요양 제도 + AI 특화의 유일한 솔루션

────────────────────────────────────────────────
2. SWOT 분석
────────────────────────────────────────────────
SWOT는 내부 역량(Strength·Weakness)과 외부 환경(Opportunity·Threat)을
교차 분석하여 전략 방향을 도출하는 프레임워크다 (교재 Part 2 Ch.10 참조).


  S (강점 — Strength)
  ──────────────────
  S1  유일한 3자 실시간 소통 구조
      보호자·의료진·환자가 하나의 앱에서 연결되는 멀티사이드 플랫폼.
      어느 한 축만 확보한 경쟁사로는 대체 불가능한 네트워크 효과 발생.

  S2  AI 일일 리포트 자동 생성 (Claude API)
      간호사가 체크리스트를 입력하면 AI가 저녁마다 자동으로 보호자용 요약 생성.
      "오늘 엄마가 어떻게 지내셨는지"를 전화 없이 매일 알 수 있다.

  S3  5개 모드 AI 케어가이드 — 핵심 차별화
      제도안내 / 건강정보 / 감정지원 / 요양원찾기 / 요양비계산
      "요양원에 대해 아무것도 모르는 보호자"를 위한 원스톱 정보 채널.
      현재 시장에 유사 서비스 없음.

  S4  역할별 완전 분리 UX
      보호자(모바일 5탭) / 의료진(PC 사이드바) / 환자(태블릿 3탭 대형 UI)
      각 역할의 실제 사용 환경에 최적화.

  S5  한국 장기요양 제도 특화
      국민건강보험공단 요양원 데이터 4,640개소, 등급별 급여 한도액, 본인부담률 등
      한국 제도에 특화된 AI 가이드 제공. 해외 서비스로 대체 불가.


  W (약점 — Weakness)
  ──────────────────
  W1  초기 단계 — 요양원 영업력 부재
      서비스 가치가 "요양원 도입" 없이는 발휘 불가능.
      의료진·환자 사용자를 확보하려면 시설 단위 영업 필요.

  W2  AI API 변동 비용
      Claude API 호출 비용이 사용량 비례로 증가. DAU 1,000명 이상 시 비용 검토 필요.

  W3  의료 데이터 민감성
      처방전, 진료기록 등 의료 정보 취급 → 법적 프레이밍 필요.
      "의료 진단" X, "케어 기록 열람"으로 포지셔닝.

  W4  팀 규모 (3인)
      기획·개발·디자인 동시 진행 → 자원 제약으로 기능 범위 제한.


  O (기회 — Opportunity)
  ──────────────────────
  O1  초고령사회 가속
      2025년 한국 공식 초고령사회 진입. 65세 이상 인구 20% 돌파.
      2027년 장기요양 이용자 145만 명 전망. 시장 자체가 연 8%+ 성장.

  O2  블루오션 시장 — 완전한 공백
      "입소 후 실시간 3자 소통" 영역 국내 공백.
      기술적으로는 최근 AI(GPT, Claude)로 비로소 구현 가능해진 영역.

  O3  정부 정책 지원
      보건복지부 AI 복지·돌봄 TF 운영. 서울형 R&D 지원사업 등 정책 기조 일치.

  O4  보호자 스트레스 급증
      심각한 스트레스 비율: 2022년 36.0% → 2024년 46.3% 급증.
      불안 강도 증가 = 솔루션 수요 증가.

  O5  글로벌 트렌드 일치
      AI 기반 노인케어 플랫폼이 글로벌 투자 핵심 세그먼트로 부상.
      Birdie(UK), Caily(US) 등 글로벌 벤치마크 존재 — 시장 검증됨.


  T (위협 — Threat)
  ──────────────────
  T1  대형 IT 기업 진입 가능성
      카카오헬스케어, 네이버클라우드 등이 노인케어 시장 진입 시 자본·브랜드 경쟁 불리.

  T2  의료 데이터 규제
      의료법 개정, 개인정보보호법 강화로 처방전·진료기록 공유 범위 제한 가능.

  T3  요양원 디지털 전환 저항
      소규모 요양원의 IT 도입 거부감. "기존 방식이 편하다"는 관성 극복 필요.

  T4  기술 의존성
      Claude API 가격 정책 변경, Supabase 서비스 장애 등 외부 의존 리스크.


  SWOT 교차 전략 (SO/ST/WO/WT)
  ──────────────────────────
  SO 전략 (강점 × 기회): 선점 전략
    → S1(3자 소통) × O2(블루오션): 국내 최초 3자 소통 플랫폼으로 시장 선점
    → S3(AI 가이드) × O1(초고령사회): AI 케어가이드로 보호자 정보 갈증 해소

  ST 전략 (강점 × 위협): 차별화 방어
    → S5(한국 제도 특화) × T1(대형 기업 진입): 제도 데이터 + 한국어 특화로 진입 장벽
    → S2(AI 리포트) × T3(디지털 저항): "보호자 전화 50% 감소" 수치로 도입 설득

  WO 전략 (약점 × 기회): 선택과 집중
    → W1(영업력 부재) × O3(정책 지원): 공모전·정부 프로그램으로 초기 파일럿 기관 확보
    → W4(팀 규모) × O2(블루오션): MVP 9개 기능 집중 완성 후 기능 확장

  WT 전략 (약점 × 위협): 리스크 관리
    → W3(의료 데이터) × T2(규제): "의료 진단 X, 케어 기록 열람" 법적 프레이밍 명확화
    → W2(AI 비용) × T4(기술 의존): 리포트 생성 주기 최적화, 캐싱 전략 적용


────────────────────────────────────────────────
3. 포지셔닝 선언 (Positioning Statement)
────────────────────────────────────────────────

핵심 포지션
  "요양원에 가족을 맡긴 보호자를 위한, AI 기반 실시간 케어 소통 플랫폼.
   입소 후 보호자·의료진·환자 3자를 하나의 앱에서 연결하여 매일의 안심을 전한다."

포지셔닝 맵 요약
  X축: 입소 전 ←→ 입소 후
  Y축: 단방향/기능 단순 ←→ 양방향/기능 다양
  하루안부 위치: 입소 후 + 양방향 + 기능 다양 = 국내 완전 공백 사분면

차별화 3가지
  ① 유일한 3자 소통: 보호자·의료진·환자 모두 연결 (경쟁사는 최대 2자)
  ② AI가 매일 자동 생성: 전화 없이도 오늘 하루를 요약해주는 AI 리포트
  ③ 원스톱 케어 정보: 5개 모드의 AI 케어가이드 (요양 정보 + 감정 지원 통합)

4. 전략 방향 결론
────────────────────────────────────────────────

단기 (W8 중간점검 기준)
  - 9개 Level 1 기능 완성
  - Figma 프로토타입으로 3자 연결 경험 시연
  - 보호자 불안 → 안심 여정 강조

중기 (W15 최종 런칭 기준)
  - 파일럿 요양원 1~2곳 도입 협의
  - 보호자 DAU 1,000명 목표
  - 전화 감소율 50% 검증

장기 (공모전·투자 유치 기준)
  - 시설 구독 B2B 모델 (월 99,000~299,000원)
  - 보호자 프리미엄 B2C (월 9,900~19,900원)
  - 데이터 누적 → AI 정확도 향상 → 해자 구축
