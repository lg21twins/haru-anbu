  03   개별 분석
3.1  토스 (Toss) — "복잡한 걸 쉽게"의 교과서
항목
분석 내용
컬러 시스템
• 메인: 토스 블루 (#3182F6) • 배경: 화이트 + 연한 그레이 • 강조: 블루 단일 포인트 — 시선이 분산되지 않음 • 7년간 운용 후 2024년 컬러 시스템 전면 개편 • 시멘틱 컬러 토큰으로 컨텍스트별 자동 변환
톤앤매너
• 미니멀, 깔끔, 신뢰감 • 화이트 스페이스(여백) 적극 활용 • 정보량은 많지만 한 번에 하나씩만 보여줌
정보 계층
• 카드 기반 레이아웃 — 정보를 카드 단위로 분리 • 큰 숫자(금액) + 작은 라벨 = 한눈에 핵심 파악 • 폰트 크기/굵기로 3단계 위계 (제목-본문-보조)
그라디언트
• 배경이나 카드에 미묘한 그라디언트 적용 • 과하지 않게 — 브랜드 컬러를 해치지 않는 선에서
하루안부 적용 포인트
★ 보호자 대시보드를 토스식 카드 UI로 설계 ★ 케어 완료율을 큰 숫자로 (예: "85%") ★ 블루 계열 포인트 컬러 + 여백 활용 ★ 시멘틱 컬러로 역할별 테마 분기

3.2  당근 — "이웃의 따뜻함"을 UI로
항목
분석 내용
컬러 시스템
• 메인: 당근 오렌지 (#FF6F0F) • 베이스: 검정/화이트 심플 배경 • 포인트: 오렌지(CTA) + 초록(후기/지도) + 청록(더보기) • 중요 정보/액션 유도 → 오렌지, 그 외 → 회색 계열
톤앤매너
• 부드러운 말투 (~요), 풀어서 설명하는 카피 • 딱딱한 어투 X → 친근하고 인간적인 느낌 • "당신 근처의 이웃" 감성 → 우리는 "가족의 안부" 감성
정보 계층
• 가격/제목 = 굵은 폰트 (사용자 관심 정보) • 설명/시간 = 연회색 (보조 정보) • 일관된 계층 구조로 시선 피로 감소
접근성
• 텍스트 크기 조절 옵션 • 고대비 색상 모드 • 음성 명령 지원 • 유니버설 디자인 원칙 적용
하루안부 적용 포인트
★ 따뜻한 말투 — "오늘 어머님은 이렇게 보내셨어요" ★ 접근성 기능 — 환자(고령자)를 위한 큰 글자/고대비 ★ 포인트 컬러로 중요 정보만 강조하는 계층 구조

3.3  밀리의서재 — "차분한 여백"의 감성
항목
분석 내용
컬러 시스템
• 메인: 차분한 베이지/크림 톤 • 포인트: 부드러운 브라운 계열 • 전체적으로 따뜻하지만 과하지 않은 톤
톤앤매너
• 감성적이고 문학적인 카피 • 과하지 않은 정보량 — "덜 보여주는" 용기 • 콘텐츠(책)가 주인공, UI는 배경 역할
레이아웃
• 적당한 간격과 여백으로 시각적 여유 • 카드 간 간격이 넓어 답답하지 않음 • 하단 탭 네비게이션 — 타 플랫폼과 동일한 구조로 학습 비용 제로
하루안부 적용 포인트
★ AI 일일 리포트를 밀리식 감성 카드로 — 여백 있고 차분하게 ★ "덜 보여주는" 원칙 — 보호자에게 필요한 핵심만 ★ 슬로건 "오늘 하루도, 안녕하셨습니다"의 감성과 일치하는 톤

3.4  카카오헬스케어 (파스타) — "건강 데이터를 시각화"
항목
분석 내용
컬러 시스템
• 메인: 부드러운 그린/블루 계열 • 건강 = 그린, 경고 = 옐로, 위험 = 레드 • 의미 기반 컬러 (시멘틱 컬러) 적극 활용
데이터 시각화
• 혈당 그래프 — 시간대별 변화를 직관적 곡선으로 • 일일 리포트 카드 — "잘한 점 / 아쉬운 점" 분리 표시 • Vision AI — 음식 촬영 → 자동 영양소 분석
AI 리포트
• CGM 데이터 → AI가 자동 분석 → 요약 리포트 • 혈당 변동성, 관리 지표, 하이라이트를 수치+그래프로 • "잘한 점"과 "아쉬운 점"을 구분해서 보여주는 UX
하루안부 적용 포인트
★ AI 일일 리포트를 파스타식 "잘한 점/아쉬운 점" 구조로 ★ 케어 완료율을 원형 그래프/프로그레스 바로 시각화 ★ 시멘틱 컬러: 정상=그린, 주의=옐로, 긴급=레드 ★ 시간대별 케어 타임라인을 곡선/바 차트로

3.5  CaringBridge — "가장 유사한 서비스"
항목
분석 내용
서비스 특징
• 미국 비영리(501c3) 건강 소통 플랫폼 • 환자 건강 업데이트를 가족/친구에게 공유 • 광고 없는 안전한 공간 — 신뢰감의 핵심
소통 UX
• 건강 업데이트 카드 — 사진 + 텍스트 + 타임스탬프 • 방명록(Guest Book) — 가족/친구가 응원 메시지 • "한 번에 모두에게" 업데이트 — 개별 연락 부담 제거
감정 지원
• 격려, 응원, 사진, 의료/케어 업데이트를   하나의 안전한 공간에서 제공 • 정보 전달 + 감정 지원이 함께 있는 구조
최근 리뉴얼
• 구세대 UI 프레임워크 → 모던 앱 UI로 전면 개편 • 리뉴얼 후 모바일 사용자 50%+ 증가 • 모바일 우선 전략이 성공
하루안부 적용 포인트
★ 건강 업데이트 카드 구조 → 우리의 "케어 리포트 카드" ★ 방명록 감성 → 보호자가 환자에게 메시지 남기는 기능 ★ "안전하고 광고 없는 공간"이라는 신뢰 메시지 ★ 모바일 우선 전략 — 보호자는 모바일이 주 사용 환경

3.6  눔 (Noom) — "따뜻한 그라디언트 + 코칭"
항목
분석 내용
컬러 시스템
• 메인: 따뜻한 옐로~오렌지 그라디언트 • 보조: 부드러운 민트/그린 • 그라디언트를 브랜드 아이덴티티의 핵심으로 활용
그라디언트 활용
• 앱 상단 배너/헤더에 부드러운 그라디언트 • CTA 버튼에 그라디언트 적용 → 시선 유도 • 과하지 않게 — 카드 배경은 화이트 유지
코칭 톤
• AI 코치가 "오늘도 잘하셨어요!" 격려 • 데이터 + 감성이 결합된 피드백 • 차가운 숫자가 아닌 따뜻한 해석을 제공
하루안부 적용 포인트
★ 그라디언트 사용법 — 헤더/배너에 부드럽게, 카드는 깔끔하게 ★ AI 케어 가이드의 톤을 눔식 코칭으로 — "오늘도 잘 돌보셨어요" ★ 블루~그린 그라디언트로 "안심"과 "케어" 이미지 결합

v2.0 업데이트: UX 패턴 벤치마킹 추가
기존 비주얼 컨셉 벤치마킹(토스, 당근, 밀리의서재, 카카오헬스케어, CaringBridge, Noom)에 더해, 레퍼런스앱 조사보고서에서 발굴된 UX 패턴 벤치마킹을 추가합니다.
이 섹션은 "어떻게 보이는가"(비주얼)보다 "어떻게 작동하는가"(인터랙션/UX 패턴)에 초점을 맞춥니다.
5.1  키즈노트 — 수신확인 + 재알림 UX
항목
분석 내용
핵심 UX 패턴
• 알림장 수신확인 시스템: 보낸 사람이 읽었는지 실시간 확인 • 미확인 시 재알림 자동 발송 • 번호 비공개 통화: 프라이버시 보호하면서 즉시 통화 가능
하루안부 적용
• F-01 체크리스트: 보호자가 확인했는지 표시 + 미확인 시 재알림 • F-03 채팅: 번호 비공개 통화 패턴 → 의료진-보호자 프라이버시
UX 원칙
"보낸 사람의 불안을 줄인다" — 수신확인은 단순 기능이 아니라 신뢰의 UI

5.2  Life360 — 서클 기반 SOS + 지오펜싱
항목
분석 내용
핵심 UX 패턴
• 서클(Circle): 가족 그룹 단위로 위치·상태 공유 • SOS 버튼: 원터치로 서클 전체에 긴급 알림 • 지오펜싱: 특정 영역 출입 시 자동 알림
하루안부 적용
• F-02 SOS: Life360식 원터치 긴급 알림 → 보호자+의료진 동시 알림 • 서클 개념 → 하루안부의 "보호자-환자-의료진" 3자 그룹과 유사
UX 원칙
"안전은 원터치" — 긴급 상황에서 복잡한 UI는 실패한다

5.3  채널톡 ALF — AI 3단계 에스컬레이션
항목
분석 내용
핵심 UX 패턴
• 1단계: FAQ 자동 답변 (즉시) • 2단계: AI가 맥락 파악 후 개인화 답변 • 3단계: 해결 불가 시 상담원에게 자동 전달 (에스컬레이션) • 사용자에게 단계 전환이 자연스럽게 느껴짐
하루안부 적용
• F-04 AI 가이드: 동일한 3단계 구조 적용   1단계: 일반 요양 정보 FAQ   2단계: AI가 환자 상태 기반 맞춤 답변   3단계: 의료진에게 자동 연결
UX 원칙
"AI는 만능이 아니다" — 한계를 인정하고 자연스럽게 사람에게 넘기는 것이 신뢰

5.4  오늘건강 — 시니어 접근성 UI
항목
분석 내용
핵심 UX 패턴
• 큰 글씨 (기본 18pt+) • 고대비 컬러 (WCAG AA 이상) • 최소한의 깊이 (2탭 이내 목표 도달) • 큰 터치 타겟 (최소 48x48dp)
하루안부 적용
• 환자(시니어) 역할 UI에 직접 적용 • 체크리스트: 큰 체크박스 + 큰 글씨 • SOS: 화면 전체를 차지하는 큰 버튼 • 전체적으로 고대비 모드 지원
UX 원칙
"접근성은 옵션이 아니라 기본값" — 시니어 UI를 별도가 아닌 기본 설계로

5.5  MyChart (Epic) — AI 기반 의료진 소통 UX
항목
분석 내용
핵심 UX 패턴
• AI/LLM이 의료진 답변 초안 자동 생성 • 의료진은 검토+수정만 → 응답 시간 단축 • 환자에게는 "의료진이 직접 쓴 것처럼" 자연스러운 메시지 • 미국 최대 환자 포털 (수천만 사용자)
하루안부 적용
• AI 가이드가 의료진 답변 초안 생성 → 의료진 업무 부담 감소 • 보호자 질문 → AI 초안 → 의료진 승인 → 보호자에게 전달 • "AI가 돕고, 사람이 확인한다" 패턴
UX 원칙
"AI는 대체가 아니라 보조" — 전문가의 판단을 AI가 가속화
