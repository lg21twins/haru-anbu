# 2부 6장: 데이터 기반 리서치

## 핵심 요약

빅데이터와 인공지능 기술의 발전으로 UX·UI 디자인에서 데이터의 중요성이 급격히 높아지고 있다. 이 장에서는 데이터 기반 디자인의 개념, 데이터 과학의 프로세스, 데이터 드리븐 디자인의 방법론, 그리고 데이터를 디자인에 활용하는 실제 방법을 다룬다.

## 1. 데이터 기반의 디자인

### 데이터 기반 디자인의 등장
- **트랜슬레이션 디자인(Translational Design)**: 종합 연구 방법으로 데이터와 디자인을 통합
- 서울시 올빼미버스 사례: KT 통신 데이터를 활용하여 심야 버스 노선 최적화
- 넷플릭스 하우스 오브 카드: 시청 데이터 분석으로 콘텐츠 기획

### A/B 테스트와 데이터 활용
- **A/B 테스트**: 두 가지 버전을 비교하여 더 나은 디자인 결정
- 데이터를 통해 **사용자 행동 패턴** 파악 및 디자인 개선
- 2,000년 이상의 성과를 보인 넷플릭스의 추천 알고리즘

## 2. 데이터 과학

### 데이터 과학이란
- 데이터 과학에서 다루는 **관계, 그 관계의 결과, 그 결과 명령의 과학적 방법론**
- 요하네스 케플러(Johannes Kepler)의 관측 데이터 분석이 과학적 접근의 시초

### 데이터의 유형
| 유형 | 설명 |
|------|------|
| **구조적 데이터(Structured)** | 수치형, 논리형, 범주형 등 체계적 형태 |
| **반구조적 데이터(Semi-structured)** | HTML, JSON, XML 등 |
| **비구조적 데이터(Unstructured)** | 텍스트, 이미지, 영상, SNS 게시물 등 |

### 데이터 과학의 프로세스 (5단계)
1. **문제 정의(Frame the Problem)**: 비즈니스 문제와 연구 질문 설정
2. **데이터 수집(Collect Raw Data)**: 원본 데이터 수집과 정제
3. **탐색적 데이터 분석(EDA, Exploratory Data Analysis)**: 데이터 특성 파악
4. **심층 모델링과 검증(Perform In-depth Analysis)**: 모델 생성과 검증
5. **결과 해석 및 배포(Communicate Results)**: 인포그래픽 등으로 시각화

### 데이터 과학과 디자인의 교차점
- Drew Conway의 **데이터 과학 벤 다이어그램**: 해킹 스킬 + 수학/통계 + 도메인 전문성
- UX 디자이너는 **도메인 전문성** 영역에서 데이터 과학과 연결

## 3. 데이터 드리븐 디자인

### 데이터 활용 수준 3단계
| 수준 | 설명 |
|------|------|
| **Data Aware(인식)** | 데이터가 존재함을 인식하고 적절한 방법 탐색 |
| **Data Informed(참고)** | API 스니펫이나 분석 목적으로 데이터 참고 |
| **Data Driven(주도)** | 통계적 방법론으로 의사결정의 근거로 활용 |

### Data-Aware 디자인
- 사용자 경험과 관리 감독에 **데이터를 인식하고 이해**
- DAU(Daily Active User), MAU(Monthly Active User) 등 핵심 지표 이해
- DIKW 모델: **데이터(Data)** → **정보(Information)** → **지식(Knowledge)** → **지혜(Wisdom)**

### Data-Informed 디자인
- 데이터를 **참조하여** 디자인 의사결정에 반영
- 기존에 쌓인 데이터로 맥락을 파악하고 **가설 수립**에 활용
- 정량적 데이터와 정성적 리서치를 결합

### Data-Driven 디자인
- 데이터가 디자인 의사결정을 **직접 주도**
- Capital One의 사례: 데이터로 디자인 프로세스 전체 운영
- Rochelle King의 **"Designing with Data"** 참조
- 더블 다이아몬드 프로세스에 데이터 기반 접근 통합

### 데이터와 디자인 관계 매트릭스
| | UX 리서치 | UI/UX·I | 사용자의 관계 |
|------|------|------|------|
| **통계 조사/연구** | 통계 리서치 | 디자이닝 | 테스트 |
| **데이터 과학** | 컨텍스트 리서치 | 솔루션 | 프로토타이핑 |
| **기업 내부 데이터** | UX 분석 | — | — |

## 4. 데이터를 디자인에 활용

### 마케팅 지표 (AARRR)
- **Acquisition(획득)** → **Activation(활성화)** → **Retention(유지)** → **Revenue(수익)** → **Referral(추천)**
- 스타트업과 테크 비즈니스에서 핵심 성과 측정 프레임워크

### 웹 분석과 사용성 테스트
- **Google Analytics(GA)**: 웹/앱 트래픽과 사용자 행동 분석
- **아이트래킹(Eye Tracking)**: 시선의 흐름과 주의 집중 패턴 추적
- **히트맵(Heat Map)**: 클릭, 스크롤, 마우스 움직임 시각화
- **클릭 맵(Click Map)**: 사용자가 마우스를 올려 놓거나 클릭한 지점 추적

### 텍스트 마이닝과 크롤링
- **웹 크롤링**: 웹에서 대량의 텍스트 데이터 수집
- **텍스트 마이닝**: 수집된 텍스트에서 의미 있는 패턴 도출
- **TF-IDF 분석**, **N-gram 분석**, **감정 분석** 등 활용
- 공공 데이터 활용: data.go.kr, data.seoul.go.kr 등

### 데이터 시각화
- **데이터 시각화 vs 인포그래픽** 차이 이해
- 데이터 시각화: **수치 데이터 기반**, 패턴·관계·추세 표현
- 인포그래픽: 복잡한 정보를 **이미지, 아이콘** 등으로 간결하게 표현
- Alberto Cairo의 데이터 시각화 가이드 참조
- 동일한 통계치의 데이터도 **시각화 방식에 따라** 전혀 다르게 보일 수 있음 (Anscombe's Quartet)

## 핵심 인사이트

1. 데이터 기반 디자인은 **Data Aware → Data Informed → Data Driven** 3단계로 성숙
2. **DIKW 모델**로 데이터에서 지혜까지의 가치 사슬을 이해
3. **AARRR 프레임워크**로 서비스 핵심 지표를 측정하고 디자인에 반영
4. **아이트래킹, 히트맵, 클릭맵** 등 행동 데이터가 UX 디자인의 근거 제공
5. **텍스트 마이닝**과 **공공 데이터** 활용으로 리서치 범위 확장
6. 데이터 시각화와 인포그래픽의 **목적과 방법의 차이**를 명확히 구분해야 함
