# 1부 5장: UX·UI 디자인 산업과 디자이너의 역할

## 핵심 요약

UX·UI 디자인은 다양한 산업 분야에서 사용자 경험을 향상시키는 중심적 역할을 하고 있다. 이 장에서는 유관 산업의 구조, 제품 서비스 시스템, 비즈니스 거래 유형, 디자이너의 역할과 필요 능력을 살펴본다.

## 1. 유관 산업과 UX·UI 디자인

### 산업 분류 체계
UX·UI 디자인 유관 산업은 **인력·프로세스·결과물** 기준으로 분류:
- **서비스 업종**: 에이전시, 플랫폼 등
- **테크**: PC·모바일·콘텐츠 플랫폼
- **미디어**: OTT·스트리밍 서비스 등
- **커뮤니티**: 소셜 미디어, 커뮤니티 플랫폼

### 산업 인프라 분류
| 계층 | 내용 |
|------|------|
| **콘텐츠/서비스** | 교육, 공공, 금융, 커머스, 게임, 미디어 등 |
| **플랫폼** | 운영체제(OS), 검색, 소셜, 커뮤니티 |
| **디바이스/하드웨어** | 모바일, PC, 웨어러블, AR/VR |
| **인프라** | 클라우드, 네트워크, 통신 |

### XR, 메타버스, XaaS
- **XaaS (Anything as a Service)**: SaaS, PaaS, IaaS 등 서비스 형태로 제공
- **PaaS (Platform as a Service)**: 공통 서비스를 플랫폼으로 제공
- 유관 서비스 산업 모델은 어떤 형태의 제품인지에 따라 변화

## 2. 제품 서비스 시스템 (PSS)

### PSS의 개념
- **PSS(Product Service System)**: 제품과 서비스의 융합으로 사용자에게 가치 제공
- UX·UI 디자인의 목표: 사용자가 원하는 결과와 상호작용을 최적의 방식으로 설계
- 2010년대부터 **결과 중심 경제(Outcome Economy)** 개념 대두

### PSS 유형
| 유형 | 설명 |
|------|------|
| 제품 지향 PSS | 제품 판매 + 부가 서비스 |
| 사용 지향 PSS | 제품 소유 대신 사용권 제공 |
| 결과 지향 PSS | 사용자에게 결과(성과)를 직접 제공 |

## 3. 비즈니스 거래 유형

### 주요 비즈니스 모델
| 유형 | 설명 | 사례 |
|------|------|------|
| **B2C** | 기업이 소비자에게 직접 판매 | 쇼핑몰, 스트리밍 |
| **B2B** | 기업 간 거래 | ERP, CRM, SaaS |
| **C2M** | 소비자→제조업체 직접 거래 | 맞춤형 제조 |
| **P2P/C2C** | 개인 간 거래 | 중고거래, 공유경제 |
| **D2C** | 기업이 유통을 거치지 않고 직접 판매 | 자체 온라인몰 |
| **B2G** | 기업→정부 | 공공 서비스 |
| **G2C** | 정부→시민 | 전자정부, 온라인 민원 |
| **B2E** | 기업→직원 | 내부 시스템 |
| **C2B** | 소비자가 기업에 가치 제공 | 리뷰, 크리에이터 |
| **B2B2C** | 기업→기업→소비자 | 리셀러, 마켓플레이스 |
| **O2O** | 온라인→오프라인 연결 | 배달 서비스 |
| **G2B** | 정부→기업 | 공공 조달 |
| **C2E** | 소비자가 직접 맞춤 제품을 요구 | 3D 프린팅, 크라우드펀딩 |

## 4. UX·UI 디자이너의 역할

### 직무별 역할
| 직무 | 설명 |
|------|------|
| **UI 디자이너** | 시각적 디자인, 인터랙션 디자인, 프로토타이핑, 컴포넌트 제작 |
| **UX 리서처** | 사용자 인터뷰, 설문 조사, 사용성 테스트 등 수행 |
| **UX 디자이너** | 사용자 리서치, 퍼소나 제작, 고객 여정 지도, 와이어프레임 등 |
| **인터랙션 디자이너** | 사용자와 제품·서비스 간의 상호작용 설계 |
| **프로토타이퍼** | 인터랙션 디자인의 시뮬레이션과 프로토타입 제작 |
| **디자인 기획자** | 전략·비즈니스 목표화의 설정 등 포함 |
| **인공지능 디자이너(AI Design)** | AI 기술을 활용해 사용자 경험 설계 |
| **데이터 기반 디자이너** | 데이터와 통계 분석을 통해 디자인 결정 내리기 |

### UX·UI 디자이너에게 필요한 능력

#### 전문적 능력
| 능력 | 세부 내용 |
|------|----------|
| **문제 해결 능력** | 규정된 및 UX를 향상시키는 능력, 아이디어로 발전시키는 능력 |
| **학습 능력** | 학습에 대한 즐거움, UX/UI 트렌드 수집과 비교 연구 |
| **공감 능력** | 다양한 사용자의 생각, 감정, 경험을 이해하고 공감 |
| **논리적 사고능력** | 정보를 분석하고, 패턴을 찾아 합리적으로 판단 |
| **표현력과 창의력** | 아이디어를 설계로 구현하는 표현력과 창의적 문제 해결 |

#### 커뮤니케이션 능력
- 아이디어/솔루션을 효과적으로 전달
- 디자이너 내부와 외부 이해관계자 간 소통
- 다양한 배경의 팀과 원활한 협업

## 핵심 인사이트

1. UX·UI 디자인 산업은 **콘텐츠/서비스 → 플랫폼 → 디바이스 → 인프라**의 계층 구조
2. 비즈니스 모델이 B2C, B2B를 넘어 **C2M, C2E, B2B2C** 등으로 다양화
3. UX·UI 디자이너는 단순 시각 디자인을 넘어 **리서치, 전략, AI, 데이터** 분야까지 역할 확장
4. **공감 능력**과 **논리적 사고**가 UX·UI 디자이너의 핵심 역량
5. 커뮤니케이션 능력이 기술적 역량만큼 중요
