# 3부 6장: 디자인 시스템과 패턴

## 핵심 요약

디자인 시스템과 패턴은 UX·UI 개발에서 일관성, 효율성, 임팩트를 제공하는 중요한 프레임워크이다. OS의 기본 개념부터 WIMP 모델, GUI 요소, 디자인 시스템의 구성, 그리고 다크 패턴과 넛지 디자인까지 다룬다.

## 1. OS의 이해

### 운영체제의 역할
- **OS(Operating System)**: 하드웨어의 리소스를 관리하고 컴퓨터 프로그램에 대해 서비스를 제공하는 시스템 소프트웨어
- 멀티태스킹, 파일 관리, 메모리 관리, 클립보드 등의 기능 제공
- UI 디자인에서는 각 운영체제의 **디자인 철학과 원칙**을 파악해야 함

### 멀티태스킹
- 운영체제가 동시에 여러 개의 작업을 처리할 수 있는 능력
- 군사적 목적에서 시작 — "Fire and Forget" 개념

## 2. GUI의 기본 — WIMP 모델

### WIMP 구성 요소
- **윈도(Window)**: 여러 애플리케이션/문서를 동시에 관리, 멀티태스킹 가능
- **아이콘(Icon)**: 파일 및 시스템 기능에 대한 바로 가기 역할, 디자인의 스타일을 좌우
- **메뉴(Menu)**: 명령과 기능에 대한 구조화된 접근 제공, 계층적으로 그룹화
- **포인터(Pointer)**: GUI의 정확한 상호작용을 위한 핵심 요소, 커서 등

### WYSIWYG
- **What You See Is What You Get**: 화면에서 보는 그대로 출력/인쇄
- 2D, 3D, VR, AR 디자인을 위한 대부분의 소프트웨어가 WYSIWYG 방식
- 와이어프레임 단계에서 바로 프로토타입을 만드는 것도 WYSIWYG

### 운영체제별 디자인 원칙
- 각 운영체제가 어떤 **철학과 원칙**을 가지고 있는지 파악해야 함
- iOS, 안드로이드 등 각각의 GUI는 고유한 디자인 가이드라인 보유

## 2. 디자인 시스템과 패턴

### 디자인 시스템의 정의
- **재사용 가능한 UI 구성 요소의 모음**이자 조합으로, 특정 제품이나 브랜드 전반에 걸쳐 임팩트를 줌
- 색상 팔레트, 타이포그래피, 레이아웃 같은 요소를 표준화하여 다양한 플랫폼과 프로젝트에서 **시각적 일관성** 유지

### 디자인 시스템의 구성
| 디자인 시스템 | 스타일 가이드 |
|------|------|
| 정보 구조, 디자인 요소 유형, 모션 접근성, 컨텐츠 방침 | 색상, 아이콘, 그리드, 레이아웃, 타이포그래피, 지간, 행간 등 |
| | **컴포넌트 라이브러리**: 990이상, 리스트, 버튼, 이미지, 아이콘, 텍스트, 내비게이션, 셀러이 등 |

### 디자인 시스템 vs UI 가이드라인
- **디자인 시스템**: 원칙적이고 권고적인 성격이 강한 반면
- **UI 가이드라인**: 실무에서 바로 사용할 수 있는 도구와 자산을 포함하는 더 큰 범위

### 디자인 시스템의 문제점
- 색상의 일관성 부족, 가독성 문제, 테두리 및 그림자 오남용
- 텍스트 길이와 레이아웃의 부재 등

### 대표적 디자인 시스템
- **Google Material Design**, **IBM Carbon Design System**, **TOSS TDS(Toss Design System)**
- **Fluent(Microsoft)**, **Atomic System** 등

## 3. 디자인 패턴

### 패턴의 개념
- 디자이너는 오랜 **축적된 해결 방식**을 정의한 패턴 랭귀지와 건축의 중첩 솜루선을 통해 다양한 상황에서 반복적으로 발생하는 문제를 해결
- UX 패턴은 데이터 처리, 입력 방식, 탐색, 조작, 알림, 화면 옮겨짐, 생체 기반, 혼합현실, 맥락 인지 등의 카테고리로 분류

### UI 패턴 라이브러리
- **ixpatterns.com** 메인 페이지의 경우, UX 패턴을 하위 항목에서 카테고리와 함께 패턴을 체계적으로 정리

## 4. 다크 패턴과 넛지 디자인

### 다크 패턴(Dark Pattern)
- 사용자의 선택을 **의도적으로 속이거나 불리하게 만드는** UX·UI 디자인
- 2010년 영국의 디자이너 해리 브리그널(Harry Brignull)이 명명
- 사례: '숨은 갱신', '최소 단위 해지', '검색한 상품의 가격이 올라가는 순차 공개 가격 책정', '사용자가 무심코 가치 유도해 원치 않는 가입을 하게 만드는 특정 옵션 사전 선택' 등
- 2023년 공정거래위원회는 소비자 피해를 유발하는 13개 행위를 도출

### 넛지 디자인(Nudge Design)
- 행동경제학에서 '합리적 인간'으로 설명하기 어려운 인간의 비합리성을 설명하는 이론
- 리처드 탈러(Richard Thaler)와 캐스 선스타인(Cass Sunstein)이 제안한 **'넛지(Nudge)'** 개념
- 사용자의 자유로운 선택을 방해하지 않으면서 **긍정적인 방향**으로 유도
- 디자인 원칙: **자유로운 선택 보장, 기본 옵션의 최적화, 사용자에게 불필요한 복잡성 제거, 피드백 제공**

### 넛지 디자인의 네 가지 요약
1. "자유로운 선택 보장"
2. "기본 옵션의 최적화"
3. "단순하고 직관적인 정보 제공"
4. "피드백 제공"

### 넛지 디자인의 중요성
- AI 기술이 발전함에 따라 사용자의 행동 데이터를 바탕으로 더욱 **개인화된 넛지 전략** 적용 가능
- 투명하고 윤리적인 디자인을 통해 사용자와의 **신뢰 관계** 구축이 궁극적 목표

## 핵심 인사이트

1. **WIMP 모델**(윈도, 아이콘, 메뉴, 포인터)은 GUI의 기본이며 운영체제별 디자인 원칙 이해가 필수
2. **디자인 시스템**은 재사용 가능한 UI 구성 요소의 모음으로 일관성과 효율성을 동시에 달성
3. **디자인 패턴**은 반복적으로 발생하는 UX 문제에 대한 축적된 해결 방식
4. **다크 패턴**은 사용자를 속이는 비윤리적 디자인 — 공정거래위원회 차원에서 규제 대상
5. **넛지 디자인**은 사용자의 자유로운 선택을 보장하면서 긍정적 방향으로 유도하는 윤리적 접근
