# 1부 2장: UX·UI 디자인 개념 형성과 특징

## 핵심 요약

UX·UI 디자인은 기술의 발달과 밀접하게 연결되어 있으며, 디지털 제품과 서비스의 설계에서 필수적인 역할을 담당한다. 이 장에서는 UX·UI 디자인의 기원과 전개, 제품 개념의 확장, CX/서비스/HCI와의 관계, 학문적 위치를 살펴본다.

## 1. UX·UI 디자인의 기원과 전개

### UI 디자인의 역사적 발전
| 시대 | 주요 기술/사건 |
|------|------------|
| 1940년대 | ENIAC(1946), 트랜지스터(1948) |
| 1960년대 | 라이트펜(1961), 스케치패드(1963), HMD(1967), 마우스(1968) |
| 1970~80년대 | 제록스 알토(1978), 제록스 스타(1981), 매킨토시(1984), MS 윈도우(1985) |
| 1990년대 | ARPANet(1969) → WWW 서비스 시작(1992), 모자이크(1993), 야후(1994) |
| 2000년대 | J-SH04(2000), iPod(2001), iPhone(2007), iPad(2010), 갤럭시 노트(2011) |
| 2020년대 | GPT-2(2019), GPT-3(2020), ChatGPT(2022), Gemini/Sora(2024) |

### UI 인터페이스의 진화
- **펀칭 카드(Punching Card)** → **명령줄(Command Line)** → **GUI** → **터치(Touch)** → **내추럴 UI(Natural UI)**
- GUI 스타일 변화: **스큐어모피즘(Skeuomorphism, 2007)** → **플랫 디자인(Flat Design, 2013)** → **뉴모피즘(Neumorphism, 2020)**

### UX 개념의 형성
- 1960년 Lawrence Rethfeld의 『The Measure of Man』: 인간공학 관점의 기초
- 1986년 Donald Norman의 『User Centered System Design』
- 1988년 『The Design of Everyday Things』 출간
- 1993년 Donald Norman이 Apple에서 **"User Experience Architect"** 직군 최초 사용
- 2005년 Don Norman의 『Emotional Design』
- 2007년 Bill Moggridge의 『Designing Interactions』

### UX Design M&A (대기업의 디자인 회사 인수)
- 2012년 Deloitte → Doblin 인수
- 2013년 Accenture → Fjord 인수
- 2014년 Capital One → Adaptive Path 인수
- 2015년 McKinsey → Lunar 인수
→ 대기업들이 UX 디자인 회사를 인수한 것은 UX가 비즈니스 전략의 핵심이 되었음을 방증

## 2. 제품 개념 확장과 UX·UI 디자인

### 제품의 확장된 정의
- 전통적: 물리적 형태를 가진 유형의 상품
- 현대적: 대량생산 소비재뿐 아니라 **소프트웨어, 서비스, 디지털 콘텐츠**까지 포함
- 제품 디자이너의 역할도 **기능을 조작하기 위한 UI, 조작의 과정에서 사용자 간의 인터랙션(Interaction)** 설계로 확장

### 제품과 UX의 관계
- 제품 = 하드웨어 + 소프트웨어 + 서비스의 통합체
- 제품 디자이너의 역할: 사용자의 감성을 고려한 경험을 설계하는 역할로 확장
- **"UX를 통해 제품과 시스템을 디자인하는 채널(으)로 접근"**

### 솔루션(Solution)의 개념
- 기업의 사업 영역에서 개인 고객 니즈에 맞춤화된 서비스를 제공하는 것
- 제품 + 서비스 + 사용자 경험을 통합한 **솔루션 제공**이 현대 디자인의 방향

## 3. CX, 서비스, HCI와 UX·UI 디자인

### CX (Customer Experience) vs UX (User Experience)
- **CX**: 고객이 브랜드/기업과의 모든 접점에서의 총체적 경험 (마케팅, 영업, 서비스 포함)
- **UX**: 사용자가 제품이나 서비스를 **사용하는 과정**에서의 경험
- CX가 더 넓은 범위, UX는 제품/서비스 사용 맥락에 집중

### 서비스 디자인(Service Design)
- UX와 서비스 디자인의 유사성이 많지만, **서비스 디자인**은 서비스 전체 과정에서 이해관계자, 접점, 프로세스를 포괄적으로 설계
- UX 디자인은 사용자 중심, 서비스 디자인은 **사용자 + 서비스 제공자** 모두 포함

### HCI (Human-Computer Interaction)
- 사용자와 시스템을 통해 연구하는 목적을 달성하기 위한 학문
- HCI는 **인간 중심의 기술과 인문학적 관점**을 결합해 사용자 경험 연구
- 컴퓨터 과학, 인지과학, 디자인, 사회학 등 **학제 간 연구(Interdisciplinary)**
- HCI → UX 디자인의 학문적 기반

### 관련 분야의 관계도
```
사용성 공학(Usability Engineering)
  ↓
사용자 인터페이스(User Interface) → 사용자 경험 디자인(UXD) → 서비스 디자인(Service Design)
  ↑                                                           ↓
HCI(Human Computer Interaction)                          고객 경험(CX)
```

## 4. UX·UI 디자인의 학문적 위치

### 학제 간 분야
- UX·UI 디자인은 **다학제적(Interdisciplinary)** 분야
- 관련 학문: 디자인 이론, 인간공학, 심리학, 과학/과학 등
- **이해하기(Understanding)와 만들기(Making)** 두 축을 기반으로 함
- Herbert Simon의 "Hard Science" 영역과도 연결

### 학문적 뿌리
- 사회학, 심리학 → 커뮤니케이션/정보디자인 → UXD → 정보통신/IT/SW
- 산업디자인 → 인지디자인 → 인터랙션 디자인
- 철학 디자인 → 인지심리 → 인지과학

### UX·UI 디자인의 학문적 특성
- **경험과학**과 **디자인학**의 교차점
- 기술과 인문학의 융합 (기초과학 + 공학프로그래밍 + 인문학적 관점)
- 사용자를 관찰하기 위해 **사회과학적 방법론** 적용 (에스노그래피 등)
- 인지심리학과 인간공학의 이론적 기반 위에 디자인 실무를 구축

## 핵심 인사이트

1. UX·UI 디자인은 기술 발전과 함께 진화해왔으며, AI/LLM 시대에 새로운 전환점을 맞이하고 있음
2. 대기업들의 디자인 회사 인수(M&A)는 **UX가 비즈니스 전략의 핵심**이 되었음을 증명
3. 제품의 개념이 물리적 상품에서 **소프트웨어+서비스+경험의 통합체**로 확장
4. CX > 서비스 디자인 > UX > UI 순으로 범위가 좁아지지만, 모두 **사용자 중심 설계**라는 공통 기반
5. UX·UI는 **학제 간(Interdisciplinary) 분야**로, 단일 학문으로 환원 불가
