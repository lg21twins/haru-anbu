# 1부 3장: UX·UI 디자인 정의 및 모델

## 핵심 요약

UX·UI 디자인을 이해하기 위해 UX, UI, 인터랙션, 사용성의 개념을 구체적으로 정의하고, 이들의 관계를 모델로 구성하여 UX·UI 디자인의 본질을 파악한다.

## 1. UX 개념 정의

### UX란?
- 국제표준화기구(ISO): **"사용자 경험은 제품, 서비스, 있는 시스템에 속하거나 그것을 사용함으로써 얻어지는 개인의 반응과 지각"**으로 정의
- UX는 사용자 중심의 **감각·인식을 패턴화하여** 통합적으로 다루는 분야

### UX의 범위
- **감정적 반응**: 사용자가 제품이나 서비스, 시스템을 사용하기 전, 중, 후에 느끼는 감정
- **사용성(Usability)**: 목표를 달성하는 데 얼마나 빠르고 효과적으로 달성할 수 있는지
- **효율성**: 사용자가 제품이나 서비스, 시스템을 사용하면서 느끼는 단순함
- 사용자 경험에 대한 정의에 **공통적으로 포함되는 요소들**을 다층적으로 정리 가능

### Peter Morville의 UX 허니콤 모델 (Honeycomb Model)
사용자 경험의 핵심 요소 7가지:

| 요소 | 영문 | 설명 |
|------|------|------|
| 유용성 | Useful | 특정한 문제를 해결하거나 필요를 충족시키는 것 |
| 사용성 | Usable | 유용한 제품이나 서비스가 단순히 존재하는 것 그치지 않고 사용하기 쉬운 것 |
| 신뢰성 | Credible | 소프트웨어에 기반 제품이나 시스템의 신뢰성과 안전 |
| 검색 가능성 | Findable | 사용자가 원하는 것을 쉽게 찾을 수 있는 것 |
| 접근성 | Accessible | 다양한 사용자가 접근 가능한 것 |
| 매력성 | Desirable | 감정적으로 매력적인 것 |
| 가치 | Valuable | 중심에 위치, 모든 요소의 통합적 결과 |

## 2. UI 개념 정의

### UI란?
- **사용자 인터페이스(User Interface)**: 사용자가 시스템과 상호작용하는 접점
- 다중 사용자, 다중 시스템 인터페이스 시대: 여러 사용자가 하나의 인터페이스를 공유하거나 한 사용자가 여러 시스템을 사용

### 인터페이스의 유형
| 구분 | 상호작용 형태 | 사례 |
|------|-----------|------|
| Human-Human (H-H) | User↔User | 대면 커뮤니케이션(face to face) |
| Human-Machine (H-M) | User↔System | CSCW, 멀티모달 인터페이스, 유비쿼터스 |
| Machine-Machine (M-M) | System↔System | 시스템 간 자동화 연결 |

### UI의 분류
- **물리적 사용자 인터페이스(PUI)**: 하드웨어, 리모컨, 다이얼 등
- **그래픽 사용자 인터페이스(GUI)**: 아이콘, 메뉴, 버튼 등 시각적 요소
- **내용적 사용자 인터페이스**: 정보 구조, 워크플로우 등을 통한 콘텐츠 설계
- **사용자 인터페이스**: 정보 아키텍처(IA), 인터랙션 워크플로우 등

## 3. 인터랙션 개념 정의

### 인터랙션이란?
- **인터랙션(Interaction)**: 인터페이스와 함께 사용되지만, 상호작용에 관한 보다 넓은 개념
- 사용자는 자신의 의미와 목표에 따라 비교한 후 선택하고, 제품 및 서비스의 목표를 달성하고 사용자와 제품, 서비스 사이에 목표를 달성하기 위해 기능을 제공하고 제공받는 **과정적 의미의 인터랙션**
- 사회적 맥락에서의 상호작용: **언어뿐만 아니라 그 이면의 사회적 맥락**을 포함

## 4. 사용성 개념 정의

### 사용성이란?
- **사용성(Usability)**: 사용자가 제품이나 시스템을 **효과적(Effectiveness)**, **효율적(Efficiency)**, **만족스럽게(Satisfaction)** 사용할 수 있는 정도
- ISO 정의: "사용자가 목표를 이루기 위해 정확한 정보를 사용하는 시스템에 대한 효과성, 효율성, 만족도"

### Nielsen의 사용성 속성 (Attributes of System Acceptability, 1993)
1. **배우기 쉬움(Easy to Learn)**: 처음 사용하는 사용자도 쉽게 학습
2. **효율성(Efficient to Use)**: 학습 후 높은 생산성 달성
3. **기억 용이성(Easy to Remember)**: 재사용 시 쉽게 기억
4. **오류 최소(Few Errors)**: 오류 발생 빈도 낮고 회복 용이
5. **주관적 만족(Subjective Pleasing)**: 사용자에게 만족스러운 경험

### 시스템 수용성의 구조
```
시스템 수용성(System Acceptability)
├── 사회적 수용성(Social Acceptability)
└── 실용적 수용성(Practical Acceptability)
    ├── 유용성(Usefulness)
    │   ├── 효용성(Utility)
    │   └── 사용성(Usability)
    ├── 비용(Cost)
    ├── 호환성(Compatibility)
    ├── 신뢰성(Reliability)
    └── 기타
```

## 5. 사용성 휴리스틱 (Jakob Nielsen의 10가지)

| # | 원칙 | 설명 |
|---|------|------|
| 1 | 시스템 상태의 가시성 | 현재 상태를 적절한 피드백으로 알려줌 |
| 2 | 실제 세계와의 일치 | 사용자 친숙한 단어와 개념 사용 |
| 3 | 사용자 제어와 자유 | 실수 시 쉽게 되돌릴 수 있는 기능 제공 |
| 4 | 일관성과 표준 | 플랫폼/산업 관례를 따름 |
| 5 | 오류 예방 | 오류 발생을 사전에 방지하는 설계 |
| 6 | 회상보다 인식 | 기억에 의존하지 않고 정보를 눈에 보이게 제공 |
| 7 | 유연성과 효율성 | 초보자와 전문가 모두를 위한 설계 |
| 8 | 미니멀리스트 디자인 | 불필요한 정보를 제거한 깔끔한 인터페이스 |
| 9 | 오류 인식, 진단 및 복구 지원 | 오류 메시지를 평문으로 표현, 해결책 제안 |
| 10 | 도움말과 설명서 | 필요 시 쉽게 접근 가능한 도움말 |

## 6. UX·UI 디자인 모델

### 모델의 핵심 구성 요소
- **사용자(User)**: 심성 모형(Mental Model)을 가지고 시스템과 상호작용
- **심성 모형(Mental Model)**: 사용자가 시스템을 이해하고 사용하는 방식에 대한 내재된 인지적 프레임워크
- **사회적 맥락(Social Context)**: 환경적, 문화적, 사회적 요인
- **상황(Situation)**: 시간, 장소, 사용자의 상태
- **시스템/제품(System/Product)**: 사용자가 상호작용하는 대상
- **인터페이스(Interface)**: 사용자와 시스템의 매개체

### UX·UI 디자인 모델 구조
```
사용자 → 요구사항/목표 → 과업/사용 → 인터페이스 → 기능/서비스
  ↕                                    ↕
심성 모형                          시스템/제품 → 네트워크 → 정보
  ↕                                    ↕              → 디바이스
사회적 맥락                           상황           → 콘텐츠/서비스
        ← 인터랙션 →
```

### 심성 모형의 중요성
- 사용자가 가진 **다양한 심성 모형을 이해하고 반영**한 UX·UI 디자인이 필요
- 디자이너가 이해하는 심성 모형 vs 사용자의 심성 모형 간 **갭을 최소화**하는 것이 핵심
- 친숙한 메타포와 패턴을 활용하면 사용자의 심성 모형에 부합하는 설계 가능

## 핵심 인사이트

1. UX는 감정·사용성·효율성을 모두 포괄하는 **통합적 경험 개념**
2. Peter Morville의 허니콤 모델: 유용성, 사용성, 신뢰성, 검색 가능성, 접근성, 매력성, 가치 7가지 축
3. Nielsen의 **사용성 휴리스틱 10가지**는 UI 평가의 기본 프레임워크
4. **심성 모형(Mental Model)** 이해가 좋은 UX·UI 설계의 핵심
5. 사용성은 단순히 "쓰기 쉬운" 것이 아니라 **효과성·효율성·만족도의 조합**
