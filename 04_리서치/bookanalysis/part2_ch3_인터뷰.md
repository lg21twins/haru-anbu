# 2부 3장: 인터뷰

## 핵심 요약

인터뷰는 UX 리서치에서 가장 효과적이고 널리 사용되는 방법이다. 인터뷰의 특징, 진행 과정, 진행자의 태도, 인터뷰 종류, 정리 방법을 체계적으로 다루며, 좋은 인터뷰와 나쁜 인터뷰의 차이를 구체적 사례로 설명한다.

## 1. 인터뷰의 특징

### 인터뷰의 강점
- 참가자의 **생각, 감정, 경험**을 깊이 이해할 수 있는 방법
- 비언어적 반응(표정, 행동)을 관찰하며 풍부한 맥락 파악
- 사전에 기억하지 못했던 **새로운 경험과 관점** 발견 가능
- **문맥적 상황**에서 제품/서비스 경험을 이해

### 질문 유형
| 유형 | 설명 |
|------|------|
| **폐쇄형 질문(Closed Questions)** | 예/아니오로 답변, 가능한 답이 제한됨 |
| **개방형 질문(Open-Ended Questions)** | 풍부하고 다양한 답변 유도 |

### 인터뷰의 범위
- 인터뷰 요소를 **확장하거나 축소**하여 조절 가능
- 현장 방문형, 전화, 화상 등 다양한 형태
- 유형에 따라 **정보의 깊이와 구성**이 달라짐 (글 → 말 → 표현 → 몸짓 → 행동)

## 2. 인터뷰의 진행

### 3단계 구조
| 단계 | 세부 단계 | 내용 |
|------|----------|------|
| **준비 단계** | 준비(Preparation) | 인터뷰 목적과 목표 설정 |
| **본 인터뷰** | 도입(Introduction) | 인터뷰어 자기소개, 목적 설명 |
| | 분위기 형성(Warm-up) | 래포(Rapport) 형성 |
| | 일반적 질문(General Issue) | 일반적인 질문으로 시작 |
| | 심층 질문(Deep Focus) | 심화된 질문 전개 |
| | 회고(Retrospective) | 인터뷰 내용 확인, 추가 보완 |
| | 마무리(Wrap-up) | 감사 인사, 후속 조치 안내 |
| **분석 단계** | 분석(Analysis) | 녹취 전사, 코딩, 분석적 방법론 적용 |

### 인터뷰 시간과 흐름
- 소요 시간: **60분~90분** 정도가 적절
- 본격적인 진행: 도입 → 분위기 형성 → 일반적 질문 → 심층 질문 → 회고 → 마무리
- **도입 단계**: 자기소개, 인터뷰 목적 설명, 녹취 동의 등
- **분위기 형성(Warm-up)**: **래포(Rapport)** 형성이 핵심 (존중, 신뢰, 감정적 연결)
- **심층 질문(Deep Focus)**: 비구조화된 질문으로 자유로운 응답 유도
- **회고(Retrospective)**: 인터뷰 내용 확인과 보충
- **마무리(Wrap-up)**: 감사 인사, 후속 서비스 안내

## 3. 인터뷰 진행자의 태도

### 핵심 원칙
- 참가자를 **긴장하게 하지 않고**, 열린 자세로 경청
- 선입견과 편견 없이 인터뷰 진행
- 참가자의 답변에 **지나치게 관여하지 않기**
- 인터뷰 본래의 **목적 의식**을 유지하면서 계획적으로 사고

### 5 Whys 기법
- 근본 원인을 파악하기 위해 **"왜?"를 5번 반복**하는 방법
- 도요타 자동차의 사키치 오노(Sakichi Ohno)가 1970년대 개발
- 예시: 앱 사용 불만족 → 로딩 시간이 길다 → 데이터가 많이 몰린다 → 초기 화면에 모든 데이터를 불러온다 → 로드맵도 변경 필요
- **표면적 문제에서 근본 원인**으로 파고드는 체계적 접근

### 나쁜 인터뷰 유형
1. **무례한 인터뷰**: 참가자를 압도하거나 위축하게 하는 태도
2. **답변에 영향을 주는 인터뷰**: 유도 질문, 편견이 담긴 질문
3. **대화가 끊어지는 인터뷰**: 폐쇄형 질문만 사용하여 단답식 응답
4. **참가자에게 부담을 주는 인터뷰**: 전문용어 사용, 해결책 요구

## 4. 인터뷰의 종류

### 전문가 인터뷰 vs 사용자 인터뷰
- **전문가(Expert User)**: 해당 분야의 전문 지식 보유, 깊이 있는 인사이트 제공
- **사용자(Extreme User)**: 극단적 사용자로부터 혁신적 기능 발견 가능
- 참가자 유형: **부정적 사용자 → 빈도적 사용자 → 공학적(전문적) 사용자** 등

### 포커스 그룹 인터뷰 (FGI)
- **FGI(Focus Group Interview)**: 보통 **6~10명** 규모의 그룹 인터뷰
- **FGD(Focus Group Discussion)**와 구분 필요
- 장점: 다양한 관점의 의견 수집, 참가자 간 상호작용
- 단점: 사전 질문지 필요, 해당 주제에 대한 전문가 필요, 환경 관리 어려움

### 비대면 인터뷰
- 코로나19 이후 비대면 인터뷰 활용 증가
- **스마트폰, PC, 태블릿** 등 다양한 기기로 원격 진행
- 물리적 제약 없이 다양한 참가자와 **시간과 장소에 구애받지 않고** 진행

## 5. 인터뷰의 정리 방법

### 어피니티 다이어그램 (Affinity Diagram)
- 인터뷰 데이터를 **주제별로 그룹핑**하여 패턴 발견
- 포스트잇을 활용한 시각적 정리
- **이너리터 다이어그램**과 **마인드맵** 등 다양한 시각화 활용

### 코딩 스킴 (Coding Scheme)
- 인터뷰 결과를 **코드(Code)**로 분류하여 체계적으로 분석
- 개방 코딩: 자유로운 카테고리 생성
- **카테고리별 코드 분류** → 빈도 분석 → 패턴 도출
- 예시: 브랜드(R-Qu), 제품/서비스(R-Cf, R-Pf), 사용자(R-Be), 시장(R-Ma) 등

### 도미넌트 분석
- 인터뷰 분석 방법 중 비교적 간결한 방식
- FGI에서 **68%의 빈도**를 보이는 도미넌트 방식과 **40% 빈도**의 FGI 비교 분석
- 토피컬 정리 방법과 병행하여 활용

### 인터뷰의 시각화
- 인터뷰 결과를 **다이어그램, 차트** 등으로 시각화
- A, B, C 참가자별 인터뷰 내용을 종합하여 **인사이트** 도출
- 결과를 프로젝트에 대한 핵심 의사결정에 활용

## 핵심 인사이트

1. 좋은 인터뷰의 핵심은 **래포(Rapport) 형성**과 **개방형 질문**
2. **5 Whys 기법**으로 표면적 문제를 넘어 근본 원인까지 도달
3. 유도 질문, 폐쇄형 질문, 전문용어 사용은 **나쁜 인터뷰**의 대표적 유형
4. **FGI**는 다양한 관점을 수집하지만 전문 진행자와 환경 관리가 필수
5. **코딩 스킴**과 **어피니티 다이어그램**으로 체계적인 데이터 정리와 패턴 도출
