# 3부 3장: 롤플레잉과 서비스 블루프린트

## 핵심 요약

롤플레잉과 서비스 블루프린트는 더블 다이아몬드 모델의 발전 단계에 해당하는 방법론이다. 롤플레잉은 신체적 활동을 통해 인터랙션을 체험하고, 서비스 블루프린트는 서비스의 전체 과정을 제공자 관점에서 시각화한다.

## 1. 롤플레잉

### 롤플레잉의 특징
- 시나리오와 달리 **신체적 활동**(동작, 대화 등)을 통해 인터랙션을 체험
- 시간 흐름 속에서 진행되며, 고객 여정 지도나 시나리오와 달리 **몸을 움직이는 렌즈** 방식
- 현장감을 통해 **감정이입**이 가능하고 즐기면서 진행 가능
- 다만 어색한 분위기에서 진행하기 어렵고, 열린 분위기 필요

### 롤플레잉과 인지적 시찰법
- **인지적 시찰법(Cognitive Walkthrough)**: 롤플레잉보다 경험 프로토타이핑이나 바디스토밍과 더 유사
- 정보 디자인에 중요한 핸사이드나 디지털 제품에 적용 가능
- 시나리오나 프로토타이핑이 갖춰진 상태에서 진행되는 롤플레잉은 **사용성 테스트와 유사**

### 병원 휠체어 개선 디자인 사례
- 대학 UX 프로젝트 수업에서 병원을 방문 조사한 결과, **5층 복도 동선이 불편**한 점 발견
- 학생들이 환자, 보호자, 간호사 역할을 맡아 **역할극을 연행**
- 바퀴가 달린 강의실 의자를 휠체어로 가장하여 시뮬레이션
- 참가자의 역할과 특성에 따라 **롤플레잉 방법을 다르게 운영**

### 레고 시리어스 플레이(LSP: LEGO Serious Play)
- 창의적 아이데이션과 롤플레잉의 유용한 도구
- 레고 그룹에서 개발한 놀이, 구성주의, 상상력이라는 **세 가지 연구 이론** 기반
- 4~12명의 참가자와 한 명의 **퍼실리테이터**로 구성된 그룹 워크숍

### LSP의 4단계 프로세스
1. **질문하기**: 퍼실리테이터가 워크숍 목적을 고려해 참가자에게 질문/도전 과제를 줌
2. **구성하기**: 참가자가 레고 브릭을 이용해 자신의 모델을 제작, 내러티브를 통해 의미 부여
3. **공유하기**: 참가자 대화를 통해 자신의 이야기를 공유하고 모델에 대해 의미 부여
4. **반영하기**: 퍼실리테이터와 다른 참가자가 이야기를 듣고 질문

### 롤플레잉 사례 — 미들마일(Middle Mile)
- 트럭 운전사가 하차장에 입차해서 출차하기까지의 과정을 롤플레잉
- 레고를 이용해 하차장을 그리고 그 위에 **트럭 운전사의 행동을 롤플레잉**
- 과정을 통해 사전 조사에서 정확히 알지 못했던 내용들을 파악

## 2. 서비스 블루프린트

### 개념
- **서비스 블루프린트(Service Blueprint)**: 서비스가 어떻게 설계될 것인지를 나타내는 설계도
- 건축이나 기계 분야에서 쓰이는 **대형 도면을 묘사**한 것에서 유래
- 서비스의 전체 과정을 **제공자 관점**에서 시각화

### 역사
- 1984년 린 쇼스탁(Lynn Shostack)이 구두닦이 서비스 과정을 블루프린트로 표현
- 하버드비즈니스리뷰(HBR)에 「Designing Services That Deliver」라는 논문으로 유래
- 구두닦이의 각 단계 소요 시간(30초, 45초, 15초 등)까지 기록

### 구성 요소
| 계층 | 설명 |
|------|------|
| **물리적 증거물(Physical Evidence)** | 고객이 접하는 물리적 요소 |
| **고객 행동(Customer Action)** | 고객의 행동과 선택 |
| **터치포인트** | 고객과 서비스의 접점 |
| **전방 영역 종업원 활동(Frontstage)** | 고객 눈에 보이는 종업원 활동 |
| **후방 영역 종업원 활동(Backstage)** | 고객에게 보이지 않는 내부 활동 |
| **지원 절차(Support Process)** | 서비스를 지원하는 시스템과 프로세스 |

### 가시선(Line of Visibility)
- 고객에게 **보이는 영역**과 **보이지 않는 영역**을 구분하는 선
- 쇼스탁의 구두닦이 블루프린트에서 처음 도입

### 디지털화에 따른 변화
- 디지털화로 인해 전통적인 서비스 블루프린트의 양식만으로 **표현하기에 충분하지 않게 됨**
- 테이블오더, 키오스크, SNS 등 **디지털 터치포인트** 추가
- 앱과 IT 요소를 물리적 요소와 **통합적으로 표현** 필요
- 쏘카의 스마트키 제공 시 쏘카존 내 라이팅 어포던스를 제공하는 아이디어를 블루프린트에 반영한 사례

## 핵심 인사이트

1. 롤플레잉은 **몸으로 체험**하는 방식으로 시나리오만으로 발견하기 어려운 인사이트를 도출
2. **레고 시리어스 플레이(LSP)**는 레고를 매개로 추상적 사고를 구체화하는 강력한 도구
3. 서비스 블루프린트는 **제공자 관점**에서 Frontstage/Backstage를 구분하여 서비스 전체를 설계
4. **가시선(Line of Visibility)**으로 고객에게 보이는/보이지 않는 영역을 명확히 구분
5. 디지털 전환에 따라 블루프린트에 **앱, IoT, SNS 등 디지털 터치포인트**를 통합 반영해야 함
