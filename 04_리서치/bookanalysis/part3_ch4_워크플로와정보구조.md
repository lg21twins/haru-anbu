# 3부 4장: 워크플로와 정보 구조

## 핵심 요약

워크플로와 정보 구조는 UI에 가까운 디자인 방법이다. 워크플로는 시스템의 각 사용 단계를 나타내고, 정보 구조는 전체 그림과 흐름을 결정하는 설계도 역할을 한다. 메뉴 설계와 메뉴 구조에 필요한 키스톤에 대해 살펴본다.

## 1. 인터랙티브 시스템의 구성 요소

### 시스템 계층 구조
- **시스템** → **서비스(기능)** → **워크플로(메뉴 구조)** → **스크린**
- 하나의 인터랙티브 시스템은 여러 서비스로 구성
- 사용자는 서비스라는 **추상적 개념**이 아니라 화면이나 소리와 같은 감각을 통해 시스템을 경험

### 서비스와 스크린숏
- 시스템의 화면, 즉 **스크린숏(Screen shot)**은 인터랙션에서 가장 비중이 높은 구성 요소
- 하나의 서비스는 많은 스크린숏으로 구성됨
- 스크린숏은 **페이지**라는 용어로 대체할 수 있음

### 정보 구조
- 하나의 시스템 안에 있는 서비스 또는 기능을 **조직화하고 구조화**한 것
- 사용자가 쉽게 이해하고 탐색하도록 하는 작업
- UX·UI 디자인 분야에서는 **"정보 구조"**라고 부름

## 2. 워크플로

### 정의
- 한 개의 태스크는 순차적인 조작 절차로 나타낼 수 있는데, 이를 **워크플로(Workflow)**라고 함
- 특정 목표를 달성하기 위해 **업무나 작업이 어떻게 흐르는지**를 정의한 것
- 1분에서 간략하게 신청만 하면 되는 것부터 복잡한 프로세스까지 포함

### 워크플로의 추상성 수준
- **높은 추상성**: 텍스트, 도형 및 연결선 (간단한 워크플로)
- **낮은 추상성**: 스크린숏에 가까운 워크플로
- 상황에 따라 높은/낮은 추상성의 워크플로가 쓰일 때가 있음

### 유저플로와 해피 패스
- **유저플로(User Flow)**: 사용자와 시스템 간의 상호작용을 설계할 때 사용자 입장에서 바라본 워크플로
- **해피 패스(Happy Path)**: 사용자가 목적 달성까지 가장 이상적인 경로

### 쏘카 사례
- "쏘카와 KTX"로 타 도시에 도착한 후 쏘카를 사용하는 서비스 예약 과정을 워크플로로 시각화
- '결제하기' 화면은 여러 메뉴를 통해서도 접근 가능 → 워크플로는 **선형 구조가 아닌 정보 구조**

## 3. 정보 구조의 유형

### 여섯 가지 유형
| 유형 | 특징 |
|------|------|
| **선형(Sequential/Linear)** | 순차적으로 배열, ATM 인출, 인터넷 쇼핑 결제 등 |
| **그리드형(Grid)** | 2차원 이상으로 배열, 스마트폰 홈스크린, 데이터베이스 |
| **계층형(Hierarchical/Tree)** | 계층적 질서로 배치, 메뉴 구조, 조직도 |
| **그물형(Mesh)** | 무제한적 연결, 페이스북·인스타그램 같은 SNS |
| **링형(Ring)** | 순환형, 미술관 동선, 지하철 2호선 |
| **스타형(Star)** | 중심 노드에서 방사형으로 뻗어 나감, 공항 허브 |
| **하이브리드형(Hybrid)** | 여러 유형을 결합, 대부분의 실제 시스템 |

### 정보 구조 선택 시 고려 요소
- **사용자의 요구**: 가장 직관적인 구조 선택
- **정보의 유형**: 복잡도와 상호 연관성
- **유연성과 확장성**: 초기 설계 이후에도 확장 가능해야 함
- **효율성**: 사용자가 최소한의 클릭이나 탐색으로 목표 도달

### 스마트폰 이후의 변화
- 스마트폰 등장 이후 **정보 구조의 개인화**가 확대
- 사용자가 홈 화면을 자유롭게 구성할 수 있게 되면서 정보 구조 설계의 의미 변화
- 사용자의 **주도적 참여와 개인화된 경험**을 중심으로 발전

## 4. 메뉴 설계

### 메뉴 설계 과정
1. **사용자 요구 분석**: 무엇을 원하는지, 어떤 방식으로 탐색하는지 파악
2. **기능/서비스의 개발·제품 측면**: 가격 포지셔닝, 판매 포인트, 인터페이스 수단의 다양성
3. **정보 구조 설계**: 논리적이고 체계적으로 구성
4. **라벨링·UX 라이팅**: 사용자가 이해하기 쉬운 용어 사용
5. **내비게이션 시각 설계**: 사용자 중심의 접근 방식

### 설계 원칙
- 넓고 얕은 구조가 좁고 깊은 구조보다 더 바람직
- 상위 메뉴 간의 균형을 맞추되 대체적으로 넓게
- 그룹화와 구조화를 통해 메뉴를 설계

## 핵심 인사이트

1. 워크플로는 사용자의 **태스크 수행 경로**를 시각화하며, 추상성 수준에 따라 다양한 형태로 표현
2. 정보 구조의 **여섯 가지 유형**(선형, 그리드, 계층, 그물, 링, 스타)을 이해하고 적절히 결합(하이브리드)
3. 정보 구조 설계 시 **사용자 요구, 정보 유형, 유연성, 효율성**을 종합적으로 고려
4. 스마트폰 이후 정보 구조는 **개인화와 사용자 주도성** 중심으로 진화
5. 메뉴 설계는 **넓고 얕은 구조**가 바람직하며, UX 라이팅과 라벨링이 중요
