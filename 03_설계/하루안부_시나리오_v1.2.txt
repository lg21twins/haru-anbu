1. 시나리오 개요
시나리오는 사용자가 서비스를 이용하는 구체적인 상황과 맥락을 서사적으로 기술한 것이다. 단순한 기능 나열이 아니라, 사용자의 목표·감정·행동을 시간 순서로 따라감으로써 실제 서비스 경험을 시뮬레이션할 수 있다 (UX 설계 교재 Ch.2 참조). 하루안부는 보호자·의료진·환자라는 세 역할이 하나의 플랫폼 위에서 서로 다른 맥락으로 만나는 서비스이므로, 각 역할의 대표 시나리오를 설계하여 기능 간 연결과 감정 흐름을 검증한다.

시나리오 작성 목적
사용자의 실제 이용 맥락과 감정 흐름을 파악한다.
기능 간 연결이 자연스러운지 검증한다.
개발 시 우선순위를 결정하는 기준을 제공한다.
팀 내 공통된 서비스 경험 이미지를 형성한다.

보호자 시나리오: AI 에스컬레이션
배경: 김미영 보호자, 어머니 혈압이 평소보다 높다는 리포트를 보고 AI에게 질문. AI 응답만으로 불안이 해소되지 않아 의료진에게 직접 연결되는 과정.

단계
상황
하루안부 역할
Step 1 AI 1차 응답
김미영이 AI 가이드에서 질문한다. "어머니 혈압이 150/95인데 괜찮을까요?"
AI가 1차 응답: "고혈압 관리 가이드에 따르면 수축기 140 이상은 주의가 필요합니다. 현재 복용 중인 약물과 함께 안정을 취하시면..."  → "해결되셨나요?" 버튼 표시
Step 2 AI 추가 답변
김미영이 "아니오"를 누르고 추가 질문한다. "약을 바꿔야 하는 건 아닌가요? 어제도 비슷했거든요"
AI가 추가 답변: "2일 연속 높은 혈압이 관찰되고 있습니다. 담당 의료진과 상담을 권장드립니다."  → "해결되셨나요?" + "의료진에게 직접 물어보고 싶어요" 버튼 표시
Step 3 의료진 연결
김미영이 "의료진에게 직접 물어보고 싶어요"를 선택한다.
담당 간호사 박지현의 채팅방으로 자동 전환. AI 대화 요약이 자동 첨부됨: "[AI 대화 요약] 보호자가 환자 혈압(150/95, 2일 연속)에 대해 질문. 약물 변경 필요 여부 상담 희망"  → 간호사가 맥락 파악 후 바로 답변 가능

기대 효과: AI가 1차/2차로 필터링하여 의료진에게 도달하는 질문 수 50% 감소. 의료진 연결 시에도 AI 요약으로 맥락 전달 → 응답 시간 단축.
보호자 시나리오: 처치 동의 디지털화
배경: 김순자 어르신이 폐렴으로 흡인 치료가 필요한 상황. 의사가 처치를 등록하면 AI가 보호자에게 쉬운 설명을 보내고, 보호자가 앱에서 전자 서명으로 동의하는 과정.

단계
상황
하루안부 역할
Step 1 처치 등록
담당 의사가 "흡인 치료(Suctioning)" 처치를 등록한다.
하루안부가 처치 등록을 감지. AI가 의료 용어를 쉬운 말로 자동 변환 시작.
Step 2 AI 설명 전송
김미영 보호자에게 알림이 도착한다.
AI가 쉬운 설명 알림 전송: "어머님의 폐렴 치료를 위해 기도에 쌓인 분비물을 빼내는 시술을 진행하려 합니다. 소요 시간: 약 10분 통증: 경미한 불편감 있을 수 있음 목적: 호흡을 편하게 하기 위함"
Step 3 설명 확인
김미영이 알림을 열어 설명을 읽는다. "아, 이런 시술이구나... 필요하겠다"
설명 카드 하단에 "동의하기" 버튼 표시. 추가 궁금한 점은 "의료진에게 질문하기" 버튼으로 채팅 연결 가능.
Step 4 전자 서명 동의
김미영이 "동의합니다"를 터치하고 전자 서명을 입력한다.
전자 서명 저장 → 의료진에게 "보호자 동의 완료" 알림 전송. 타임라인에 기록: "2026.03.23 14:32 보호자 김미영 흡인 치료 동의 완료"  → 의사는 바로 시술 진행 가능.

기대 효과: 보호자 의료 행위 이해도 향상, 불필요한 치료 거부 감소, 동의 절차 디지털화로 행정 시간 절감, 의료진-보호자 신뢰 구축.
