1. 화면 명세서 개요
────────────────────────────────────────────────
본 문서는 하루안부 v6 목업의 전체 화면을 기록한 공식 명세서다.
각 화면의 목적, 주요 구성 요소, 내비게이션 연결, 관련 기능(PRD F-번호),
데이터 소스를 정의하여 개발·디자인·기획 간 공통 기준으로 활용한다.

전체 화면 수 : 26개
역할 구분
  - 온보딩 (공통)   : 3개 (ob01~ob03)
  - 보호자 (guardian) : 10개 (g01~g10)
  - 간호사 (nurse)    : 10개 (n01~n10)
  - 환자 (patient)    : 3개  (p01~p03)

파일 위치 : 07_디자인/mockup/v6_전체화면/
공통 CSS  : common.css (디자인 토큰 + 컴포넌트 라이브러리)

3. 온보딩 화면 명세 (공통)
────────────────────────────────────────────────

[ob01] 스플래시 (ob01-splash.html)
목적       앱 첫 진입 시 브랜드 아이덴티티 표시, 인증 상태 자동 확인
주요 UI
  - 브랜드 그라디언트 배경 (#4A7FBF → #5BAE8C)
  - 하루안부 로고 + 슬로건 "오늘 하루도, 안녕하셨습니다"
  - 하단 로딩 인디케이터
자동 이동
  - 인증 있음 → 역할에 따라 g01 / n01 / p01
  - 인증 없음 → ob02 로그인
관련 기능  F-07 역할 분기 + 인증
데이터     Supabase Auth 세션 토큰 확인
표시 시간  약 1.5초

[ob02] 로그인 (ob02-login.html)
목적       소셜 로그인으로 사용자 인증
주요 UI
  - 하루안부 로고 + 브랜드 그라디언트 배경
  - "카카오로 시작하기" 버튼 (카카오 옐로우)
  - "구글로 시작하기" 버튼 (화이트)
  - 서비스 이용약관 동의 텍스트
자동 이동
  - 로그인 성공 + 역할 있음 → g01 / n01 / p01
  - 로그인 성공 + 최초 가입 → ob03 역할 선택
관련 기능  F-07 역할 분기 + 인증
데이터     Supabase Auth (카카오/구글 OAuth)

[ob03] 역할 선택 (ob03-role.html)
목적       최초 가입 시 사용자 역할 설정
주요 UI
  - 3개 역할 카드 (보호자 / 의료진 / 환자)
  - 각 카드: 역할 아이콘 + 이름 + 한 줄 설명
  - 보호자: 파란 계열 / 의료진: 초록 계열 / 환자: 베이지 계열
  - 선택 시 카드 강조 + "시작하기" 버튼 활성화
자동 이동  역할 저장 → g01 / n01 / p01
관련 기능  F-07 역할 분기 + 인증
데이터     users 테이블 role 컬럼 업데이트

[g01] 보호자 홈 (g01-home.html)
목적       보호자 메인 화면. 환자의 오늘 케어 현황을 한눈에 파악
탭         탭1 (홈)
스크롤 구성 (위 → 아래)
  1. Status bar (9:41 / 배터리 / 와이파이)
  2. 인사 영역: 날짜 + "안녕하세요, OO님"
  3. 긴급 알림 배너 (있을 때만 표시) → g06 링크
  4. 히어로 케어 카드 (Apple Card 스타일 파란 그라디언트)
     - 환자 이름 + 요양원명 + 완료율 링(73%) + 케어 상태 요약
  5. 스탯 칩 2×2 그리드: 식사 / 투약 / 위생 / 활동
  6. 담당 간호사 카드: 프로필 + "바로 연락하기" 버튼 → g03
  7. 오늘 일정 타임라인 (3건 미리보기)
  8. 빠른 메뉴 2×2 그리드 (AI 가이드 / 리포트 / 처방전 / 채팅)
  9. 최근 활동 타임라인 (4건)
  10. AI 일일 리포트 요약 카드 → g02
  11. AI 케어 가이드 슬라이더 (가로 스크롤 카드 5장) → g04
  12. 탭바
내비게이션
  - 긴급 알림 배너 탭 → g06
  - 간호사 카드 "연락하기" → g03
  - AI 리포트 카드 → g02
  - AI 가이드 슬라이더 카드 → g04
  - 탭바 탭2 → g03 / 탭3 → g04 / 탭4 → g02 / 탭5 → g05
관련 기능  F-01(체크리스트), F-02(대시보드), F-03(SOS), F-05(AI리포트), F-06(AI가이드)
데이터     care_records, alerts, patients, messages, daily_reports

[g02] AI 일일 리포트 (g02-ai-report.html)
목적       AI가 생성한 오늘 하루 케어 요약 리포트 상세 확인
탭         탭4 (리포트)
스크롤 구성
  1. 헤더: "AI 일일 리포트" + 날짜
  2. 히어로 카드 (흰 배경): 총평 한 줄 + 케어 점수 3칩 (식사/투약/활동)
  3. AI 요약 텍스트 (자연어 서술, Claude API 생성)
  4. 항목별 상세 타임라인
     - 식사: 아침/점심/저녁 각 섭취율 + 메모
     - 투약: 약명 + 시간 + 완료 여부
     - 활동: 재활, 산책 등 시간 + 내용
     - 특이사항: 간호사 메모 (있을 때만)
  5. "읽음 확인" 버튼
  6. 탭바
관련 기능  F-05 AI 일일 케어 리포트
데이터     daily_reports, care_records
API        Claude API (리포트 텍스트 생성)

[g03] 채팅 (g03-chat.html)
목적       담당 간호사와 1:1 채팅
탭         탭2 (채팅)
화면 구성
  - iMessage 스타일 헤더: 간호사 아바타 + 이름 + 상태
  - 채팅 버블 (수신: 흰 배경 / 발신: 파란 배경)
  - 날짜 구분선
  - 하단 입력바: 텍스트 + 사진 첨부 + 전송 버튼
내비게이션  헤더 뒤로가기 → g01
관련 기능  F-04 1:1 채팅
데이터     messages 테이블 (Supabase Realtime)

[g04] AI 케어 가이드 (g04-ai-guide.html)
목적       AI 챗봇으로 요양·건강 정보 질문, 감정 지원
탭         탭3 (AI 가이드)
화면 구성
  - 상단 모드 칩 5개 (가로 스크롤): 제도안내 / 건강정보 / 감정지원 / 시설안내 / 자유질문
  - 채팅 UI: 인사 메시지 + 사용자 입력 + AI 응답 버블
  - AI 응답: 카드형 (절차 목록, 체크리스트, 링크 포함 가능)
  - 에스컬레이션 버튼: "의료진에게 연결" (3단계 미해결 시)
  - 하단 입력바
관련 기능  F-06 AI 케어 가이드 (5개 모드)
데이터     ai_chat_sessions
API        Claude API (모드별 System Prompt)

[g05] 더보기 (g05-mypage.html)
목적       부가 기능 메뉴 모음 + 케어 대상 정보
탭         탭5 (더보기)
스크롤 구성
  1. 그라디언트 프로필 카드: 보호자 이름 + 아바타 + 가입일
  2. 케어 대상 카드: 환자 이름 + 요양원 + 등급 + 입소일
  3. 기능 메뉴 그룹
     - 케어 정보: 처방전 (→g09) / 의료 타임라인 (→g10)
     - 계정 설정: 알림 설정 (→g07) / 결제/청구 (→g08)
     - 고객 지원: 공지사항 / 도움말 / 로그아웃
  4. 앱 버전 정보
관련 기능  F-08(시간표), F-09(처방전/타임라인)
데이터     users, patients

[g06] 긴급 알림 상세 (g06-alert.html)
목적       SOS 긴급 알림 내용 확인 및 대응
진입        알림 배너 탭 / 푸시 알림 탭
화면 구성
  - 빨간 히어로 배너: "긴급 알림" + 시간 + 환자명
  - 상황 설명 카드: 간호사 메모 내용
  - 대응 안내 카드: AI가 제안하는 대응 방법
  - 경과 타임라인: 발송 → 수신 → 확인 → 대응 완료
  - "확인했습니다" 버튼 (큰 버튼, 파란색)
  - 간호사 채팅 바로가기 버튼 → g03
내비게이션  확인 버튼 탭 후 → g01 / 채팅 버튼 → g03
관련 기능  F-03 SOS 긴급 알림
데이터     alerts 테이블

[g07] 알림 설정 (g07-settings.html)

  - iOS 스타일 토글 스위치 (ON: 초록, OFF: 회색)
관련 기능  알림 설정 (Supabase 사용자 설정)
데이터     users 테이블 notification_settings

[g08] 결제/청구 (g08-billing.html)

  2. 오늘 현황 요약 칩 3개: 체크리스트 완료율 / SOS 건수 / 미읽은 채팅
  3. 담당 환자 카드 리스트 (환자명 / 방번호 / 케어 완료율 / 특이사항)
  4. 미완료 알림 섹션 (투약 미완 / 체크리스트 미완)
  5. AI 인수인계 바로가기 배너 → n06
  6. 탭바
내비게이션
  - 환자 카드 탭 → n02 환자 상세
  - AI 인수인계 배너 → n06
  - 탭바 탭2 → n02 / 탭3 → n03 / 탭4 → n04
관련 기능  F-01(체크리스트), F-02(대시보드), F-03(SOS)
데이터     patients, care_records, alerts, messages

[n02] 환자 관리 (n02-patients.html)
목적       담당 환자 목록 + 개별 환자 상세 접근
탭         탭2 (환자)
화면 구성
  - 상단 검색바 + 필터 칩 (전체 / 중요 / 주의)
  - 환자 카드 리스트
    · 환자 이름 / 방번호 / 요양등급 / 오늘 케어 진행률
    · 상태 뱃지: 정상(초록) / 주의(노랑) / 긴급(빨강)

  - 하위 메뉴 진입: 채팅(→n07) / 시간표(→n08) / 처방전(→n09) / 타임라인(→n10)
관련 기능  F-01(체크리스트), F-08(시간표), F-09(처방전)

[n03] 체크리스트 (n03-checklist.html)

화면 구성
  - 상단 환자 선택 드롭다운
  - 카테고리 탭: 식사 / 투약 / 위생 / 활동
  - 체크 항목 카드 (아침/점심/저녁 또는 시간대별)
    · 체크 버튼 + 수치 입력 (식사량 %, 체온 등)
    · 메모 입력 필드 (선택)
    · 타임스탬프 자동 저장
  - 저장 버튼 → Supabase Realtime으로 보호자 대시보드 반영
관련 기능  F-01 케어 체크리스트
데이터     care_records (INSERT/UPDATE), Supabase Realtime

[n05] SOS 알림 센터 (n05-sos.html)

  - 고대비, 단순 구조 (화면당 기능 1~2개)

[p01] 환자 홈 (p01-home.html)
목적       오늘 일과 확인 + 다음 일정 강조
탭         탭1 (홈)
화면 구성
  1. 인사: "안녕하세요, 순자 어르신" + 오늘 날짜 (큰 글자)
  2. 다음 일정 카드 (가장 크게): 시간 + 내용 + 남은 시간
  3. 오늘 일과 타임라인
     · 완료 항목: 체크 표시(✅) + 회색
     · 진행 중: 초록 강조
     · 예정 항목: 일반 표시
  4. 탭바
관련 기능  F-08 시간표 (환자 열람)
데이터     schedules 테이블

[p02] 가족 소식 (p02-family.html)
목적       가족이 보낸 사진/메시지 확인 + 처방전 간단 확인
탭         탭2 (가족)
화면 구성
  - 가족 메시지 카드: 보호자 이름 + 메시지 텍스트 + 사진 (큰 글자)
  - 사진 첨부 시 전체 너비 표시
  - "처방약 확인" 섹션: 현재 약명 + 복용 시간 (간단히, 큰 글자)
관련 기능  F-04(채팅 읽기), F-09(처방전 열람, 환자 권한)
데이터     messages, prescriptions (본인 읽기만)

[p03] 도움 요청 (p03-help.html)
목적       원터치 도움 요청 (큰 버튼)
탭         탭3 (도움)
화면 구성
  - 화면 전체에 3개 큰 버튼 (각 버튼 전체 너비)
    · "물이 필요해요" 버튼 (파란 계열) → 요양보호사 알림
    · "추워요" 버튼 (주황 계열) → 요양보호사 알림
    · "아파요" 버튼 (빨간 계열, 크고 강조) → 간호사 긴급 알림
  - 버튼 탭 후 → "요청이 전달되었어요" 확인 메시지 (3초 후 자동 닫힘)
관련 기능  SOS 경량 버전 / 환자 도움 요청
데이터     help_requests 테이블 → alerts 연동
