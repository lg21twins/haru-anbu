## 2. 역할 분담 (3인 체계)

> 구체적인 역할은 팀 내 협의 후 확정. 아래는 권장 구조.

| 역할 | 담당 영역 | 비고 |
|------|-----------|------|
| **팀원 A** (프론트엔드 리드) | React UI 컴포넌트, 역할별 화면 구현, 반응형 레이아웃 | PC 웹 + 모바일 반응형 |
| **팀원 B** (백엔드/DB 리드) | Supabase 스키마, RLS 정책, Realtime, Auth, Storage | API 로직 + 보안 |
| **팀원 C** (AI/통합 리드) | Claude API 연동, AI 케어 가이드, 일일 리포트, FCM 푸시 | AI 기능 + 배포 |

> 모든 팀원이 Claude 코드를 활용하여 서로의 영역을 보조합니다.

## 3. 15주 상세 로드맵 (수업 타임라인 반영)

> 수업 5단계: 기획(1-3) → 디자인(4-7) → 중간점검(8) → 구현 및 정교화(9-14) → 최종 런칭(15)

### Phase 1: 기획 (Week 1~3)

```
Week 1 ─────────────────────────────────────────────
├── 트렌드 조사, 시장 현황 분석
│   ├── 초고령사회 진입 데이터 수집
│   ├── 장기요양 이용자·급여비용 통계
│   └── 보호자 스트레스 지표 수집
├── PRD v4.0 최종 검토 및 확정
└── 서비스 네이밍 결정

Week 2 ─────────────────────────────────────────────
├── 경쟁사 분석 보고서 완성 ✅ (완료)
│   ├── 국내 6개 서비스 (케어닥, 케어네이션, 케어포, 엔젤시스템 등)
│   ├── 해외 6개 서비스 (CaringBridge, CareSmartz360 등)
│   └── 시장 공백 도출 + 차별화 전략 수립
└── 아이디어 구체화 (AI 케어 가이드 5개 모드 설계)

Week 3 ─────────────────────────────────────────────
├── User Flow 설계 (3역할 × PC/모바일)
│   ├── 보호자 Flow: 로그인 → 대시보드 → 채팅/알림/AI가이드
│   ├── 의료진 Flow: 로그인 → 체크리스트 → SOS/채팅/처방전
│   └── 환자 Flow: 로그인 → 홈 → 일과표/처방전
├── 기능 명세서 확정 (Level 1/2/3 구분)
├── DB 스키마 설계 (11개 테이블)
├── AI 케어 가이드 프롬프트 설계 초안
└── 개발 환경 세팅 (React + TypeScript + Tailwind)
```

**산출물**: PRD 최종본, 경쟁 분석 보고서, User Flow, DB 스키마, AI 프롬프트 초안

### Phase 2: 디자인 (Week 4~7)

```
Week 4 ─────────────────────────────────────────────
├── 디자인 시스템 정의
│   ├── 컬러 팔레트 (3역할별 테마 컬러)
│   ├── 타이포그래피 (Pretendard 기반)
│   ├── 컴포넌트 라이브러리 (버튼, 카드, 인풋, 모달)
│   └── 아이콘 세트
└── 정보구조도(IA) 구체화
    ├── 전체 화면 맵 (3역할 × 기능별)
    └── 네비게이션 설계 (PC 사이드바 / 모바일 탭바)

Week 5 ─────────────────────────────────────────────
├── 무드보드 · 스타일 가이드 확정
├── 와이어프레임 — 의료진 PC 화면
│   ├── 체크리스트 입력, SOS 발송, 채팅
│   ├── 시간표/스케줄, 처방전·진료기록 입력
│   └── 환자 목록, 의료 타임라인
└── Figma 목업 시작

Week 6 ─────────────────────────────────────────────
├── 와이어프레임 — 보호자 모바일 화면
│   ├── 대시보드, AI 케어 가이드 채팅, 1:1 채팅
│   ├── AI 일일 리포트, 처방전·진료기록 열람
│   └── 시간표, SOS 알림 수신
├── 와이어프레임 — 환자 모바일/태블릿 화면
│   ├── 환자 홈 (큰 글자 UI)
│   ├── 일과표 확인, 처방전 확인
│   └── UX 카피라이팅 (안내 문구, 에러 메시지, 온보딩)

Week 7 ─────────────────────────────────────────────
├── Figma 목업 완성 (전체 화면)
├── 인터랙션 프로토타입 (주요 플로우)
├── 팀 내부 디자인 리뷰 및 피드백 반영
└── 중간점검 발표 자료 준비
```

**산출물**: 디자인 시스템, 정보구조도, Figma 프로토타입, 무드보드

### Phase 4: 구현 및 정교화 (Week 9~14)

```
Week 9 ─── 기반 구축 ──────────────────────────────
├── React + TypeScript + Tailwind 프로젝트 생성
├── Supabase 프로젝트 생성 (11개 테이블 + RLS)
├── Vercel 배포 파이프라인 구축
├── F-07 인증: 카카오·구글 소셜 로그인 + 역할 분기
└── 3역할 레이아웃 구축 (PC 사이드바 / 모바일 탭바)

Week 10 ─── 핵심 기능 1 ───────────────────────────
├── F-01 케어 체크리스트
│   ├── 의료진 PC: 환자별 식사/투약/위생/활동 체크
│   └── 타임스탬프 자동 저장 + Realtime 구독
├── F-02 보호자 대시보드
│   ├── 오늘 케어 현황 카드형 + 실시간 반영
│   └── 체크리스트 완료율 시각화
└── F-03 SOS 긴급 알림
    ├── SOS 버튼 + 상황 메모 → 푸시 알림 (30초 이내)
    └── 미응답 시 자동 재알림 + 대응 타임라인

Week 11 ─── 핵심 기능 2 ───────────────────────────
├── F-04 1:1 채팅 (Realtime + 사진 첨부)
├── F-08 시간표 (환자 일과표 + 의료진 스케줄)
└── F-09 처방전·진료기록 + 의료 타임라인
    ├── 의료진 PC: 처방전/진료기록 입력
    ├── 보호자: 열람 + 의료 타임라인 통합 뷰
    └── 환자: 본인 처방전 확인 (큰 글자)

Week 12 ─── AI 기능 + 푸시 ────────────────────────
├── F-06 AI 케어 가이드 (핵심 차별화)
│   ├── Claude API 연동 + System Prompt 설계
│   ├── 백과사전/감정지원/요양원찾기/요양비계산 모드
│   └── 대화 세션 저장
├── F-05 AI 일일 케어 리포트 (Claude API 자동 요약)
└── FCM 푸시 알림 통합 (SOS, 채팅, 리포트)

Week 13 ─── 테스트 + 품질 관리 ────────────────────
├── Level 1 전체 기능 (9개) 통합 테스트
│   ├── 역할별 시나리오 테스트
│   └── 엣지 케이스 + RLS 검증
├── 사용성 테스트 (팀 외부 3~5명)
├── 성능 측정 (3초/30초/95% 목표)
└── 버그 수정 + UI/UX 폴리싱

Week 14 ─── 정교화 + 제출 준비 ────────────────────
├── 최종 버그 수정 + 성능 최적화
├── Level 2 UI 구현 (시간 여유 시)
├── 제출 문서 완성 (사업계획서, 기술 소개서, AI 활용 보고서)
├── 발표 자료 제작 (PPT + 데모 영상)
└── 앱 최종 검수 + Vercel 프로덕션 배포
```

**산출물**: 동작하는 완성 앱, 성능 지표 달성, 제출 문서 일체

**마일스톤**: Week 10 끝 — 체크리스트→대시보드 실시간 반영 / Week 12 끝 — AI 기능 동작

## 5. 리스크 관리

| 리스크 | 확률 | 영향 | 대응 전략 |
|--------|------|------|-----------|
| 구현 기간 부족 (6주) | 높음 | 높음 | Level 2 기능 과감히 포기, Level 1에 집중 |

| 디자인 퀄리티 부족 | 높음 | 중간 | Tailwind UI + 오픈소스 컴포넌트 적극 활용 |
| 팀원 학습 곡선 | 높음 | 중간 | Claude 채팅으로 학습 병행, 페어 프로그래밍 |
| 의료법/개인정보 이슈 | 낮음 | 높음 | 법적 프레이밍 철저 (PRD 8.3절), 면책 문구 |
